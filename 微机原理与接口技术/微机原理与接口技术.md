# 第一章：
## 小简答：
1. 计算机字长（Word）指的是什么？
	- 处理器每个**单位时间**可以处理的**二进制数据位数**称计算机字长。
2. 总线信号分成哪三组信号？
	- 总线信号分成三组，分别是**数据总线**、**地址总线**和**控制总线**。
3. PC机主存采用DRAM组成还是SRAM组成？
	- PC机主存采用**DRAM**组成。
4. Cache是什么意思？
	- **高速缓冲存储器Cache**是处理器与主存之间**速度很快**但**容量较小**的存储器。
5. ROM-BIOS是什么？
	- ROM-BIOS是“**基本输入输出系统**”，操作系统通过对BIOS的调用驱动各硬件设备，用户也可以在应用程序中调用BIOS中的许多功能。
6. 中断是什么？
	- 中断是CPU正常执行程序的流程被某种原因**打断**、并**暂时停止**，**转向执行**事先安排好的一段处理程序，待该处理程序结束后仍**返回**被中断的指令继续执行的过程。
7. 32位PC机主板的芯片组是什么？
	- 主板芯片组是主板的核心部件，它提供主板上的**关键逻辑电路**。
8. 教材中MASM是指什么？
	- MASM是微软开发的**宏汇编程序**。
9. 处理器的“取指­－译码－执行周期”是指什么？
	- **指令的处理过程**。处理器的“取指—译码—执行周期” 是指处理器从主存储器**读取指令**（简称**取指**），**翻译指令代码**的功能（简称**译码**），然后**执行**指令所规定的**操作**（简称**执行**）的过程。
10. 本课程的主要内容属于计算机系统层次结构中哪个层次？
	- **机器语言层**，即指令集结构。
## 判断题：
1. **软件与硬件的等价性原理说明软硬件在功能、性能和成本等方面是等价的。**
	- ❌
	- 软件与硬件在**功能上可以等价实现**（即同一功能既可用硬件也可用软件实现），但在**性能、成本、功耗、灵活性等方面通常并不等价**。例如，硬件实现通常更快但成本高、不灵活；软件实现灵活但速度慢。因此，“等价性”仅指功能层面，并非性能和成本等。
2. **IA-64结构是IA-32结构的64位扩展，也就是Intel 64结构。**
	- ❌
	- **IA-64**（如Itanium处理器）是Intel与HP联合开发的**全新64位架构**，与x86不兼容；而**Intel 64**（也称x86-64或x64）才是IA-32的64位扩展，兼容32位x86代码。两者完全不同。
3. **8086的数据总线为16位，也就是说8086的数据总线的个数、或说条数、位数是16。**
	- ✅
	- 8086是16位微处理器，其**外部数据总线宽度为16位**，即有16条数据线（D0–D15），可一次传输2字节数据
4. **微机主存只要使用RAM芯片就可以了。**
	- ❌
	- 主存主要由RAM（如DRAM）构成，但**还需要地址译码器、存储控制器、刷新电路**（对DRAM而言）等支持电路。此外，有些系统还包含少量ROM用于启动。仅靠RAM芯片无法构成完整主存系统。
5. **处理器并不直接连接外设，而是通过I/O接口电路与外设连接。**
	- ✅
	- 这是计算机体系结构的基本原则。I/O接口（如UART、USB控制器、南桥芯片等）负责**信号转换、速率匹配、协议处理**等，使CPU能通过标准方式访问各种外设。
6. **处理器是微机的控制中心，内部只包括5大功能部件的控制器。**
	- ❌
	- 处理器（CPU）主要包括**运算器（ALU）、控制器、寄存器组、高速缓存**（Cache）等，现代CPU还包含浮点单元、分支预测、多核等复杂结构。“只包括5大功能部件的控制器”表述错误且模糊。控制器只是CPU的一部分。
7. **Windows的模拟DOS环境与控制台环境是一样的。**
	- ❌
	- 在Windows NT系列（如XP、10、11）中，**没有真正的DOS环境**，所谓的“命令提示符”是**Win32控制台子系统**，运行的是Windows原生命令（如cmd.exe），并非DOS。
	- Windows 9x 系列（直到Windows 98）基于DOS，但之后的NT 内核（Windows NT -> 2000 ->XP -> ... -> 至今）完全脱离了DOS。
	- 控制台环境 ≠ DOS模拟器（如DOSBox）。二者在内核、API、权限、功能上均不同。
8. **16位IBM PC/AT机采用ISA系统总线。**
	- ✅
	- IBM PC/AT（1984年）引入了**16位ISA总线**（Industry Standard Architecture），是在PC/XT的8位总线基础上扩展而来，成为当时标准。
9. **IA-32处理器吸取了RISC技术特长。RISC是指复杂指令集计算机。**
	- ❌
	- 前半句正确（如Pentium Pro以后的IA-32处理器内部采用RISC-like微架构），但**RISC是指精简指令集计算机**（Reduced Instruction Set Computer），而**CISC**才是**复杂指令集**（Complex Instruction Set Computer）。题干后半句定义错误。
10. **处理器进行读操作，就是把数据从处理器内部读出传送给主存或外设。**
	- ❌
	- **读操作**是指**处理器从主存或外设读取数据到内部寄存器**；而将数据从处理器送出是**写操作**。题干把方向搞反了。
	- 与常规理解相反，计算机的读是**从外界将数据读到程序/CPU**，而写是**从程序/CPU输出数据到外界**，类推到高级语言的文件读写等读写操作也是类似的。

## 大简答：
1. **说明微型计算机系统的硬件组成及各部分作用。**
	- **CPU**：CPU也称处理器，是微机的核心。它采用大规模集成电路芯片，芯片内集成了控制器、运算器和若干高速存储单元（即寄存器）。处理器及其支持电路构成了微机系统的控制中心，对系统的各个部件进行统一的协调和控制。
	- **存储器**：存储器是存放程序和数据的部件。
	- **外部设备**：外部设备是指可与微机进行交互的输入（Input）设备和输出（Output）设备，也称I/O设备。I/O设备通过I/O接口与主机连接。
	- **总线**：互连各个部件的共用通道，主要含数据总线、地址总线和控制总线信号。

2. **什么是通用微处理器、单片机（微控制器）、DSP芯片、嵌入式系统？**
	- **通用微处理器**：适合较广的应用领域的微处理器，例如装在PC机、笔记本电脑、工作站、服务器上的微处理器。
	- **单片机**：是指通常用于控制领域的微处理器芯片，其内部除CPU外还集成了计算机的其他一些主要部件，只需配上少量的外部电路和设备，就可以构成具体的应用系统。
	- **DSP芯片**：称数字信号处理器，也是一种微控制器，其更适合处理高速的数字信号，内部集成有高速乘法器，能够进行快速乘法和加法运算。
	- **嵌入式系统**：利用微控制器、数字信号处理器或通用微处理器，结合具体应用构成的控制系统。

3. **综述Intel 80x86系列处理器在指令集方面的发展。**
	- 8086奠定了基本的**16位指令集**，80286提供了**保护方式的各种指令**，80386将指令集全面提升为**32位**，80486融入了**浮点数据处理指令**，奔腾系列陆续增加了**多媒体指令**MMX、SSE、SSE2和SSE3，最新的奔腾4处理器还支持**64位指令集**。

4. **区别如下概念：助记符、汇编语言、汇编语言程序和汇编程序。**
	- **助记符**：人们采用**便于记忆、并能描述指令功能**的符号来表示机器指令操作码，该符号称为指令助记符。
	- **汇编语言**：用助记符表示的指令以及使用它们编写程序的规则就形成汇编语言。
	- **汇编语言程序**：用汇编语言书写的程序就是汇编语言程序，或称汇编语言源程序。
	- **汇编程序**：汇编语言源程序要翻译成机器语言程序才可以由处理器执行。这个翻译的过程称为“汇编”，完成汇编工作的程序就是汇编程序（Assembler）。

5. **区别如下概念：路径、绝对路径、相对路径、当前目录。系统磁盘上存在某个可执行文件，但在DOS环境输入其文件名却提示没有这个文件，是什么原因？**
	- **路径**：操作系统以**目录**形式管理磁盘上的文件，文件所在的**分区和目录**就是该文件的路径。
	- **绝对路径**：从**根目录**到**文件所在目录**的**完整路径**称为“绝对路径”。是保证文件唯一性的标示方法。
	- **相对路径**：从系统**当前目录**到**文件所在目录的路径**称为相对路径。
	- **当前目录**：**用户当前所在的目录**就是当前目录。
	- **指明的路径不正确**，或者执行了另外一个同名的文件。

6. **什么是摩尔定律？它能永久成立吗？**
	- **每18个月，集成电路的性能将提高一倍，而其价格将降低一半。**（1965年，Intel公司的创始人之一摩尔预言：集成电路上的晶体管密度每年将翻倍。现在这个预言通常表达为：每隔18个月硅片密度（晶体管容量）将翻倍；也常被表达为：每18个月，集成电路的性能将提高一倍，而其价格将降低一半。）
	- **不能**。由于电子器件的物理极限在悄然逼近，摩尔定律不会永远持续。

7. **冯·诺依曼计算机的基本设计思想是什么？**
	- 采用**二进制形式**表示数据和指令。指令由**操作码和地址码**组成。
	- 将程序和数据存放在**存储器**中，计算机在工作时从存储器取出指令加以执行，自动完成计算任务。这就是“存储程序”和“程序控制”（简称存储程序控制）的概念。
	- 指令的执行是**顺序**的，即一般按照指令在存储器中存放的顺序执行，程序分支由转移指令实现。
	- 计算机由**存储器、运算器、控制器、输入设备和输出设备**五大基本部件组成，并规定了5部分的基本功能。

8. **计算机系统通常划分为哪几个层次？普通计算机用户和软件开发人员对计算机系统的认识一样吗？**
	- 最上层是**用户层**。
	- 第5层是**高级语言层**。
	- 第4层是**汇编语言层**。
	- 第3层是**操作系统层**。
	- 第2层是**机器语言层**。
	- 第1层是**控制层**。
	- 第0层是**数字电路层**。
	- 普通计算机用户和软件人员对计算机系统的认识并不一样。普通计算机用户看到的计算机，也就是我们最熟悉的计算机，属于**用户层**，而软件人员看到的属于**高级语言层**或是**汇编语言层**。

9. **什么是系列机和兼容机？你怎样理解计算机中的“兼容”特性？例如，你可以用PC机为例，谈谈你对软件兼容（或兼容性）的认识，说明为什么PC机具有如此强大的生命力？**
	- 系列机是指在一个厂家生产的具有相同计算机结构，但具有不同组成和实现的一系列（Family）不同档次、不同型号的机器。
	- 兼容机是指不同厂家生产的具有相同计算机结构（不同的组成和实现）的计算机。
	- 兼容是一个广泛的概念，包括**软件兼容、硬件兼容、系统兼容**等。其中软件兼容是指**同一个软件可以不加修改地运行于体系结构相同的各档机器，结果一样但运行时间可能不同。**
	- 软件兼容可从机器性能和推出时间分成**向上（向下）** 和**向前（向后）** 兼容。例如32位PC机就陆续增加了对浮点处理指令、多媒体指令等的支持。在保证向后兼容的前提下，不断改进其组成和实现，延续计算机结构的生命，才使得PC机具有如此强大的生命力。

# 第二章
## 小简答：
1. ALU是什么？
	- ALU是**算术逻辑运算单元**，负责处理器所能进行的各种运算，主要是算术运算和逻辑运算。
2. 8086的取指为什么可以被称为指令预取？
	- 取指是指从主存取出指令代码通过总线传输到处理器内部指令寄存器的过程。8086分成总线接口单元和指令执行单元，可以独立操作。**在执行单元执行一条指令的同时，总线接口单元可以读取下一条指令，等到执行时不需要进行取指了**，所以称为预取。
3. Pentium的片上Cache采用统一结构还是分离结构？
	- Pentium采用**分离的Cache结构**，一个用做**指令Cache**，一个用做**数据Cache**。
4. 堆栈的存取原则是什么？
	- 堆栈的存取原则是**先进后出**（也称为后进先出）操作方式存取数据。
5. 标志寄存器主要保存哪方面的信息？
	- 标志寄存器主要保存反映**指令执行结果**和**控制指令执行形式**的有关状态。
6. 执行了一条加法指令后，发现ZF＝1，说明结果是什么？
	- 执行了一条加法指令后，发现ZF＝1，表明**运算结果为0**。
7. 汇编语言中的标识符与高级语言的变量和常量名的组成原则有本质的区别吗？
	- **没有**。
8. 汇编语言的标识符大小写不敏感意味着什么？
	- 汇编语言的标识符大小写不敏感，即表示字母大小写不同、但表示同一个符号。
9. 汇编语言源程序文件中，END语句后的语句会被汇编吗？
	- **不会**。（**`END` 告诉汇编器：源程序到此结束，并且指定程序的入口点（即程序的起始执行地址）**）
10. 为什么将查找操作数的方法称为数据寻“址”方式？
	- 指令的操作数需要**通过存储器地址或I/O地址**，才能查找到数据本身，故称数据寻址方式。

## 判断题：
1. **程序计数器 PC 或指令指针 EIP 寄存器属于通用寄存器。**
	- ❌
	- PC（在x86中为IP/EIP/RIP）是**专用寄存器**（控制寄存器），用于存放下一条要执行的指令地址。
	- 通用寄存器（如EAX、EBX等）用于数据运算和存储，可由程序员自由使用。
	- EIP **不能被直接读写**（如 `MOV EIP, EAX` 是非法的），而通用寄存器可以。
2. **处理器的指令译码是将指令代码翻译成它代表的功能的过程，与数字电路的译码器是不同的概念。**
	- ✅
	- **指令译码**（Instruction Decode）是CPU取指后，解析操作码（opcode）以确定要执行的操作（如加法、跳转等），属于**微架构行为**。
	- **数字电路中的译码器**（如3-8译码器）是组合逻辑电路，用于地址选通等，功能单一。
	- 两者虽然都叫“译码”，但**抽象层次和用途完全不同**。
3. **EAX也被称为累加器，因为它使用最频繁。**
	- ✅
	- 在x86架构中，**EAX（AX/AL）传统上称为“累加器”**（Accumulator）。
	- 许多指令（如 `MUL`, `DIV`, `IN`, `OUT`, `LODS`, 字符串操作等）**隐式使用EAX**，因此使用频率高。
	- 这一称呼源于早期8086设计，沿用至今。
	- EAX（累加器），EBX（基地址寄存器），ECX（计数器），EDX（数据寄存器）【数据】
	- EBP（基址指针寄存器），ESP（堆栈指针寄存器），ESI（源变址寄存器）、EDI（目的变址寄存器）【指针】
4. **处理器的传送指令MOV属于汇编语言的执行性语句**。
	- ✅
	- 汇编语言语句分为两类：
	    - **指示性语句**（伪指令，如 `.data`, `DB`, `END`）——不生成机器码；
	    - **执行性语句**（指令语句，如 `MOV`, `ADD`, `JMP`）——对应真实机器指令。
	- `MOV` 会生成机器码，在运行时由CPU执行，属于**执行性语句**。
5. **汇编语言的语句由明显的4部分组成，不需要分隔符区别。**
	- ❌
	- 汇编语句通常包含4个字段：`[标号:] 操作码 [操作数] [;注释]`。
	- 但**必须用分隔符**（通常是空格或制表符）来区分各部分。例如：
		- `loop_start:  MOV  AX, BX    ; 注释`
	    - 标号后需冒号 `:`；
	    - 操作码与操作数之间需**空格**；
	    - 多个操作数之间用逗号 `,` 分隔。
	- 若无分隔符，汇编器无法正确解析。
6. **80减90（80－90）需要借位，所以执行结束后，进位标志CF＝1。**
	- ✅
	- 在x86中，**CF（Carry Flag）在无符号数运算中表示借位/进位**。
	- 对于减法：`CF = 1` 表示**被减数 < 减数**（即需要向高位借位）。
	- 80 - 90（无符号）：80 < 90 → 需要借位 → **CF = 1**。
	- ✅ 注意：这是**无符号数**视角；有符号数看的是OF（溢出标志）。

| 标志位 | 全称                   | 中文名     | 含义                                     |
| --- | -------------------- | ------- | -------------------------------------- |
| CF  | Carry Flag           | 进位/借位标志 | 无符号数运算时发生进位（加法）或借位（减法）时置 1。            |

	1. **逻辑地址 → 线性地址**：

		- CPU 用段选择子在 **GDT 或 LDT** 中查找对应的 **段描述符（Segment Descriptor）**。
		- 段描述符中包含 **32 位段基地址（Base Address）**。
		- **线性地址 = 段基地址 + 偏移地址**
	2. **线性地址 → 物理地址（取决于是否启用分页）**
		- **如果未启用分页（Paging = OFF）**：  
		    → 线性地址 = 物理地址
		- **如果启用了分页（Paging = ON，现代操作系统几乎都开启）**：

	- 梳理：“在保护模式下，段基地址（来自段描述符）加上偏移地址得到的是**线性地址**；若**未启用分页**，则线性地址等于物理地址；若**启用分页**，则需通过**页表转换**才能得到物理地址。”

# 第四章：
## 小简答：
1. **是什么特点决定了目标地址的相对寻址方式应用最多？**
    - 当同一个程序被操作系统安排到不同的存储区域执行时，指令间的位移没有改变，目标地址采用相对寻址可方便操作系统的灵活调度。
2. **什么是奇偶校验？**
    - 数据通信时，**数据的某一位用做传输数据的奇偶校验位**，数据中包括校验位在内的“1”的个数恒为奇数，就是奇校验；恒为偶数，就是偶校验。
3. **为什么判断无符号数大小和有符号大小的条件转移指令不同？**
    - **无符号数和有符号数的操作影响两组不同的标志状态位**，故判断两个无符号数和有符号数的大小关系要利用不同的标志位组合，所以有对应的两组指令。
4. **双分支结构中两个分支体之间的JMP指令有什么作用？**
    - 双分支结构中两个分支体之间的JMP指令，用于实现结束前一个分支回到共同的出口作用。
5. **为什么特别强调为子程序加上必要的注释？**
    - 完整的子程序注释可方便程序员调用该子程序，子程序注释包括**子程序名**、**子程序功能**、**入口参数**和**出口参数**、**调用注意事项**和**其他说明**等。
6. **子程序采用堆栈传递参数，为什么要特别注意堆栈平衡问题？**
    - 子程序保持堆栈平衡，才能保证执行RET指令时当前栈顶的内容是正确的返回地址。主程序也要保持堆栈平衡，这样才能释放传递参数占用的堆栈空间，否则多次调用该子程序可能就致使堆栈溢出。
7. **参数传递的“传值”和“传址”有什么区别？**
    - “传值”是传递参数的一个**拷贝**，被调用程序改变这个参数不影响调用程序；“传址”时，被调用程序可能修改**通过地址引用**的变量内容。
8. **INCLUDE语句和INCLUDELIB有什么区别？**
    - INCLUDE语句包含的是**文本文件**、是源程序文件的一部分；INCLUDELIB语句包含的是子程序库文件。
9. **混合编程有什么优势？**
    - 取长补短。
10. **运行Windows程序，有时为什么会提示某个DLL文件不存在？**
    - Windows程序在运行时需要加载其配套的动态链接库DLL文件，当其没有被搜索到时就会提示不存在。
## 判断题：
1. **指令指针或者还包括代码段寄存器值的改变将引起程序流程的改变。**
    - ✅
    - 指令指针（IP/EIP/RIP）指向下一条要执行的指令地址，代码段寄存器（CS）与之共同构成当前代码的逻辑地址。任何对 CS:IP 的修改（如通过 JMP、CALL、RET 等）都会导致程序跳转到新位置执行，从而改变程序流程。
2. **JMP指令对应高级语言的GOTO语句，所以不能使用。**
    - ❌
    - 虽然 JMP 在功能上类似于 GOTO，但“不能使用”是错误的。在汇编语言中，JMP 是实现循环、分支、函数调用等控制流的基础指令，合理使用是必要且合法的。问题在于滥用会导致代码难以维护，而非禁止使用。
3. **因为条件转移指令Jcc要利用标志作为条件，所以也影响标志。**
    - ❌
    - 条件转移指令（如 JE、JNZ、JA 等）**只读取**标志位（如 ZF、CF、SF 等）来决定是否跳转，**不会修改任何标志位**。影响标志的是之前的算术或逻辑指令（如 CMP、SUB、TEST 等）。
4. **JA和JG指令的条件都是“大于”，所以是同一个指令的两个助记符。**
    - ❌
    - JA（Jump if Above）用于**无符号数**比较，条件是 CF=0 且 ZF=0；  
    - JG（Jump if Greater）用于**有符号数**比较，条件是 (SF=OF) 且 ZF=0。  
    - 两者判断逻辑和适用场景完全不同，并非同一指令的别名。
5. **控制循环是否结束只能在一次循环结束之后进行。**
    - ❌
    - 循环可以在**开始前**（如 while）、**中间**（如 break/continue）或**结束后**（如 do-while）判断终止条件。汇编中可通过在循环体任意位置插入条件跳转实现灵活控制，并非仅限于循环末尾。
6. **介绍LOOP指令时，常说它相当于DEC ECX和JNZ两条指令。但考虑对状态标志的影响，它们有差别。LOOP指令不影响标志，而DEC指令却会影响除CF之外的其他状态标志。**
    - ✅
    - LOOP 指令先将 ECX 减 1，若结果不为零则跳转，**整个过程不改变任何状态标志**（FLAGS 不变）；

    - ✅

10. **导入库文件和静态子程序库文件的扩展名都是LIB，所以两者性质相同。**
    - ❌
    - 虽然扩展名同为 `.lib`，但：
        - **静态库**（Static Library）包含实际的函数代码，在链接时被复制进可执行文件；
        - **导入库**（Import Library）仅包含 DLL 中导出函数的符号和桩信息，用于链接阶段解析外部引用，运行时仍需对应的 `.dll` 文件。

## 小简答：
1. **为什么称处理器的数据总线是双向的？**
    - 数据总线承担着处理器与存储器、外设之间的数据交换，既可以输入也可以输出，故其是双向的。
2. **8086的地址和数据总线为什么要分时复用？**
    - 为减少引脚个数，8086采用了地址总线和数据总线分时复用。即数据总线在不同时刻还具有地址总线的功能。
3. **具有三态能力的引脚输出高阻意味着什么？**
    - 具有三态能力的引脚当输出呈现高阻状态时，相当于连接了一个阻抗很高的外部器件，信号无法正常输出；即放弃对该引脚的控制，与其他部件**断开连接**。
4. **总线周期中的等待状态是个什么工作状态？**
    - 处理器的运行速度远远快于存储器和I/O端口。处理器检测到存储器或I/O端口不能按基本的总线周期进行数据交换时，插入一个等待状态 $T_w$ 。等待状态实际上是一个保持总线信号状态不变的时钟周期。
5. **猝发传送是一种什么传送？**
    - 猝发传送是处理器只提供首地址、但可以从后续连续的存储单元中读写多个数据的一种传输方式。
6. **总线数据传输为什么要进行总线仲裁？**
    - 总线上可能连接多个需要控制总线的主设备，需要确定当前需要控制总线的主设备，所以需要总线仲裁。
7. **异步时序为什么可以没有总线时钟信号？**
    - 异步时序是由**总线握手（Handshake）联络（应答）信号**控制，不是由总线时钟控制。故总线时钟信号可有可无。
8. **32位PC机为什么采用多级总线结构，而不是单总线结构？**
    - 单总线结构限制了许多需要高速传输速度的部件。32位PC机采用多种总线并存的系统结构。各种专用局部总线源于处理器芯片总线，以接近处理器芯片引脚的速度传输数据，它为高速外设提供速度快、性能高的共用通道。
9. **USB总线有几个导线组成？**
    - 4个。
10. **什么是微软宣称的即插即用PnP（Plug-and-Play）技术？**
    - 即插即用技术是指32位PC机的主板、操作系统和总线设备配合，实现自动配置功能。

# 知识点总结：
## 1. 总线类型：
1. **芯片总线**（Chip Bus）： 
	- **芯片级互连**，大规模集成电路芯片内部，或系统中各种不同器件连接在一起的总线
	- 局部总线（Local Bus），微处理器的引脚信号 
	- 片内总线，大规模集成电路芯片内部连接 
2. **内总线**（Internal Bus） 
	- **模板级互连**，主机内部功能单元（模板）间连接的总线 
	- 板级总线、母板总线、系统总线 
	- 系统总线（System Bus）是微机系统的主要总线
	- 内部总线从一条变为多条，形成多总线结构 
3. **外总线**（External Bus） 
	- **设备级互连**，微机与其外设或微机之间连接的总线 
	- 过去指**通信总线**；现在常延伸为**外设总线**

## 2. 总线仲裁：
### （1） 概念：决定当前控制总线的主设备
### （2）分类：
1. **集中仲裁**：
	- 系统具有中央仲裁器（控制器）
	- 负责主模块的总线请求和分配总线的使用
2. **分布仲裁：**
	- 各个主模块都有自己的仲裁器和唯一的仲裁号
	- 主模块请求总线时，发送其仲裁号
	- 比较各个主设备仲裁号决定当前控制总线的主设备

## 3. 同步方式：
- **同步时序**（共用总线时钟信号、快等慢）
- **异步时序**（应答信号、启动信号、确认信号、操作周期可变）

## 4. 传输类型：
- **读**：从到主
- **写**：主到从
- **猝发传送**：给出起始地址，将固定块长的数据一个接一个地从相邻地址读出或写入
- **写后读**、**读修改写**：
	- 这俩都是对**同一个地址单元**进行的
- **广播**：一个主设备对多个从设备的写入操作

## 5. 性能指标：
- 总线宽度：决定单次传输的数据量（bits）；
- 总线频率：表示每秒传输次数（Hz）
- 总线带宽：二者乘积即为总线带宽（bit/s、）

## 6. 总线信号和时序：
### （1）按照信号分类：
- 地址总线、数据总线、控制总线
### （2）总线时序：
#### a. 定义：
- 描述总线信号随时间变化的规律以及总线信号间的相互关系
- 采用**时序图**形象化地表现时序
#### b. 8086的总线时序：
- 8086的基本总线周期为**4个**时钟周期（未插入 $T_w$ 的情况下）
	- 其实就是**4个T周期**
- **写总线周期**：
	- **T1状态**：
		- 输出20位存储器地址A19～A0
		- $M/IO$ 输出**高电平**，表示存储器操作；输出低电平表示 I/O 操作
		- $ALE$ 输出**正脉冲**，表示复用总线输出地址
	- **T2状态：**
		- 输出控制信号 $\overline{WR}$ （低有效）
		- 撤销地址，处理器在$AD_{15}$~$AD_0$上输出数据
		- 在$A_{19}$~$A_{16}$ 上输出状态信息 $S_6$ ~ $S_3$
	- **T3状态：**
		- 轮询检测 $READY$ 脚信号，若有效，进入T4；若无效，插入 $T_w$ 状态等待一个时钟周期。
		- $T_w$ 状态延续 T3 时的状态，时长为1个时钟周期。
	- **T4状态：**
		- 释放控制信号 $\overline{WR}$ 和总线 ，完成数据传送。

> - **T2状态**：输出读/写控制信号。
> - **T3状态**：锁存地址、处理器提供的控制信号和数据在总线上继续维持有效，且T3时钟的前沿（下降沿）对READY引脚进行检测。READY信号有效，进入T4周期。
> - **T4状态**：总线周期的最后一个时钟周期，处理器和存储器或I/O端口继续进行数据传送，直到完成，并为下一个总线周期做好准备。

# 第六章：
## 小简答：
1. **存储系统为什么不能采用一种存储器件构成？**
    - 因为各种存储器件在容量、速度和价格方面存在矛盾。速度快，则单位价格高；容量大，单位价格低，但存取速度慢。故存储系统不能采用一种存储器件。
2. **什么是高速命中和高速缺失（未命中）？**
    - Cache中复制着主存的部分内容。当处理器试图读取主存的某个字时，Cache控制器首先检查Cache中是否已包含有这个字。若有，则处理器直接读取Cache，这种情况称为高速命中；若无，则称为高速缺失。
3. **高速缓存Cache系统的标签存储器有什么作用？**
    - 标签存储器保存着该数据所在主存的地址信息。
4. **什么是Cache的地址映射？**
    - 主存块与Cache行之间的对应关系称“地址映射”，Cache通过地址映射确定**一个主存块应放到哪个Cache行组中**。
5. **Cache的写入策略用于解决什么问题？**
    - 写入策略用于解决写入Cache时引起**主存和Cache内容不一致性**的问题。
6. **存储器的存取时间和存取周期有什么区别？**
    - 存取时间是指**从读/写命令发出，到数据传输操作完成**所经历的时间；存取周期表示**两次存储器访问所允许的最小时间间隔**。存取周期 $\geq$ 存取时间。
7. **虚拟存储器是什么存储器？**
    - 虚拟存储器是由操作系统利用辅助存储器、以磁盘文件形式建立的、在主存储器与辅助存储器之间的一个存储器。
8. **DRAM芯片怎么有行地址又有列地址？**
    - DRAM芯片容量大、芯片小，高集成度，引脚数量少。故DRAM芯片将地址引脚分时复用，即用一组地址引脚传送两批地址。第一批地址称行地址，第二批地址称列地址。
9. **地址重复是怎么回事？**
    - 译码电路中只有部分地址线参与译码会造成地址重复，也就是一个存储单元占有多个存储器地址。
10. **页表项的P（D0）位有什么作用？**
    - 页表项的P位称为存在位（Present），表示该页面是否在物理存储器中。
## 判断题：
1. **存储系统的高速缓存需要操作系统的配合才能提高主存访问速度。**
    - ❌
    - 高速缓存（Cache）由**硬件自动管理**，其命中、替换和一致性维护均由处理器内部逻辑完成，无需操作系统干预。因此，Cache 提升主存访问速度是**纯硬件机制**，不依赖操作系统。
2. **指令访问的操作数可能是8，16或32位，但主存与Cache间却以数据块为单位传输。**
    - ✅
    - 处理器访问的数据粒度可以是字节、字或双字，但为了提高效率，Cache 与主存之间的数据交换是以“**块**”（Cache Line，通常为几十字节）为单位进行的，这是局部性原理的应用。
3. **为了加快段描述符和页表项的访问速度，IA-32处理器内部分别设置了段描述符高速缓冲器和转换后备缓冲器。它们的基本工作原理类似主存的高速缓存。**
    - ✅
    - 段描述符缓存（Descriptor Cache）和 TLB（Translation Lookaside Buffer）分别缓存段描述符和页表项，避免每次地址转换都访问内存中的描述符表或页表，其工作方式与 Cache 类似，属于专用高速缓存。
4. **存储器芯片的集成度高表示单位芯片面积制作的存储单元数多。**
    - ✅
    - 集成度是衡量半导体存储器技术水平的重要指标，集成度越高，意味着在相同面积的硅片上可集成更多存储单元，从而实现更大容量、更低成本的芯片。
5. **微机大容量主存一般采用DRAM芯片组成。**
    - ✅
    - DRAM（动态随机存取存储器）具有结构简单、集成度高、成本低的优点，适合构建大容量主存；虽然需要刷新，但在现代系统中通过内存控制器自动管理，因此被广泛用于PC主存。
6. **部分译码可以简化译码电路，不会减少可用的存储空间。**
    - ❌
    - 部分译码确实简化了译码电路，但由于未使用全部地址线进行译码，会导致多个地址映射到同一物理存储区域，造成“地址重叠”或“地址浪费”，从而**减少了实际可用的独立寻址空间**。
7. **存储系统每次给DRAM芯片提供刷新地址，被选中的芯片上所有单元都刷新一遍。**
    - ❌
    - DRAM 刷新是以**行**为单位进行的。每次提供一个行地址，仅刷新该行的所有存储单元，而不是整个芯片的所有单元。完整刷新需依次提供所有行地址。
8. **存储系统的刷新地址提供给所有DRAM芯片。**
    - ✅
    - 在同步刷新模式下，所有 DRAM 芯片在同一时刻接收相同的行地址（刷新地址），同时刷新各自对应的行，这是由内存控制器统一提供的，确保整个存储体正确维持数据。
9. **FPM DRAM芯片中的快页读写方式就是猝发传送方式。**
    - ❌
    - FPM DRAM（Fast Page Mode DRAM）的快页模式允许在同一个行地址下快速切换列地址进行多次读写，虽具有连续访问优化，但**不是真正的猝发传送（Burst Transfer）**。猝发传送通常指由首地址自动生成后续地址并连续传输多个数据，常见于 SDRAM 及以后的 DRAM 技术。
10. **ROM芯片的烧写或擦写就是指对ROM芯片的编程。**
    - ✅
    - 对于 PROM、EPROM、EEPROM 或 Flash 等可编程 ROM，"烧写"指将数据写入，“擦写”指先擦除再写入，这些操作统称为对 ROM 的编程，尽管不同类型的 ROM 编程方式和可重复性不同。

# 第七章：
## 小简答：
1. **外设为什么不能像存储器芯片那样直接与主机相连？**
    - 外部设备在工作原理、驱动方式、信息格式以及工作速度等方面彼此差别很大，与处理器的工作方式也大相径庭。所以，外设不能像存储器芯片那样直接与处理器相连，必须经过一个中间电路（即接口电路）。
2. **计算机两个功能部件、设备等之间为什么一般都需要数据缓冲？**
    - 数据缓冲用于匹配快速的处理器与相对慢速的外设或两个功能部件之间速度不匹配的数据交换，避免数据丢失或系统效率下降。
3. **什么是接口电路的命令字或控制字？**
    - 处理器向接口芯片相应端口写入特定的数据，用于选择I/O芯片的工作方式或控制外设工作，该数据称命令字或控制字。
4. **PC机中CMOS RAM属于主存空间吗？**
    - PC机中CMOS RAM不属于主存空间，是主板上用CMOS工艺制作的，以电池供电的小型RAM，一般用于存放 BIOS 。CMOS RAM有64个字节容量，以8位I/O接口形式与处理器连接，通过两个I/O地址访问。
5. **与系统总线连接的输入接口为什么需要三态缓冲器？**
    - 在采用共享数据总线的输入接口设计中，**每个可能向总线发送数据的设备都应通过三态缓冲器连接到总线**，并通过处理器的地址/控制信号进行选通，确保**同一时刻只有一个设备驱动总线**，其他设备处于高阻隔离状态，从而避免总线冲突。
6. **透明锁存器和非透明锁存器是什么区别？**
    - 透明锁存器的控制端为有效电平时，输出随输入变化，常称为直通或透明；非透明锁存器不论其控制端为低或为高电平，输出状态都不随输入变化。
    - 神经病，书里都没有非透明锁存器的概念，只有电平触发和边沿触发的概念，电平触发常称为直通或透明，边沿触发只在控制端上升/下降沿进行数据锁存，在稳定时锁存的数据不会改变
7. **什么样的外设可以采用无条件数据传送方式？**
    - 如发光二极管、按键和开关等简单设备，它们的工作方式十分简单；相对处理器而言，其状态很少发生变化或变化很慢。这些设备与处理器交换数据时，可采用无条件传送。
8. **什么是查询超时错误？**
    - 在查询程序中，当查询超过了规定的时间，设备仍未就绪时，就引发超时错误。
9. **远调用CALL指令和INT N指令有什么区别？**
    - 远调用CALL指令利用直接或间接寻址调用另一个代码段的子程序；INT n指令利用中断向量表（地址表）的方法调用另一个代码段的中断服务程序，还有保存标志寄存器的功能。
10. **为什么说外部中断才是真正意义上的中断？**
    - 外部中断是由处理器外部提出中断请求引起的程序中断。相对于处理器来说，外部中断是随机产生的，所以是真正意义上的中断。

## 基本概念：
- 微机的外部设备多种多样，工作原理、驱动方式、信息格式、以及工作速度方面彼此差别很大。它们不能与CPU直接相连，必须经过**中间电路（I/O接口）** 再与系统相连。
- **I/O接口**：位于基本系统与外设间，用于实现两者数据交换的控制电路。
	- 在PC机主板上的**可编程接口电路**
	- 系统总线插槽中的**电路卡**（Card）

### **接口电路的外部特性：**
- I/O接口位于**处理器与外设之间**，起到“桥梁”的作用。
- **面向微处理器一侧的信号**：
	- 与处理器总线或系统总线类似
	- 有数据信号、地址信号和控制信号等 
- **面向外设一侧的信号**：
	- 与外设有关
	- 外设数据信号、外设状态信号和外设控制信号 
### 基本功能：
- **数据缓冲**：匹配快速的处理器与相对慢速的外设的数据交换
- **信号变换**：把信号相互转换为适合对方的形式

### 软件编程：
- 硬件接口电路需要软件编程配合工作
- **命令字（控制字）：**
	- 为具有一定的通用性，I/O接口芯片设计有多种工作方式，处理器通过向接口芯片的对应引脚写入**命令字或控制字**，选择其工作方式。
- **初始化程序**：
	- 选择I/O接口的工作方式、设置原始工作状态等的程序。
- **驱动程序**：
	- 操纵I/O接口完成具体工作的程序
---------------
### I/O端口：
- **I/O端口＝I/O地址**，对应当前I/O接口的寄存器
	- 一个端口对应一个寄存器，但可能可以访问多种数据，比如CMOS RAM可以使用70H输入要访问的寄存器编号，CMOS将数据输出到71H对应的数据缓冲寄存器，则可以通过71H读取CMOS内部的多个配置单元的数据。
- 一个接口电路可以具有多个I/O端口，每个端口用来保存和交换不同的信息
- 数据寄存器、状态寄存器和控制寄存器占有的I/O地址常依次被称为**数据端口、状态端口和控制端口**
- **输入、输出端口可以是同一个I/O地址（复用）**
- **编址方式**：
	- **独立编址**：单独开辟一个地址空间表示I/O端口的地址，独立于存储器地址空间。
		- 空间独立，**不占用主存空间** -> 控制和译码**电路简单**
		- 需设计**独立的I/O指令、I/O引脚** -> 独立的指令使得程序中的I/O操作**一目了然，清晰易懂**
		- 由于独立编址，所以I/O指令**没有存储器指令丰富**，且**功能较为简单**。
	- **统一编址**：I/O端口地址与处理器统一编排，共享一个地址空间。
		- **不用独立设计**I/O指令和引脚，可使用**灵活**的存储器访问方式进行I/O接口的访问。
		- 由于共享地址空间，其会**占据部分存储器空间**，且由于采用存储器访问指令进行I/O操作，会导致**难以区分存储器操作和I/O操作**，**程序难以阅读**。
- **I/O译码**：跟存储器译码在原理和方法上完全相同，多采用部分译码。

### ✅I/O操作指令：
- 中心为处理器，输入输出都是针对处理器来说的；
- `IN AL/AX/EAX, i8/DX`
	- **输入指令**：数据从 I/O 接口输入到处理器
	- **源操作数**为 `8` 位立即数（直接寻址，限制256个地址）或 `16` 位寄存器`DX`（寄存器寻址，可寻址`0000H~FFFFH`的全部 65536 个 I/O 地址），表示 I/O 端口号（地址）
	- **目的操作数**为 `8/16/32` 位寄存器 `AL/AX/EAX` ，表示存放位置（只能存放到 `AL/AX/EAX` 中）
- `OUT i8/DX, AL/AX/EAX`
	- 输出指令：数据从处理器输出到 I/O 接口
	- 源操作数和目的操作数与`IN`指令相反。

### 实地址存储模型：
- 主存空间1MB：`00000H ~ FFFFFH`
- **分段管理**：
	- 段长**64KB**
	- 段的开始地址只能位于**低4位地址全为0**的物理地址处（如`80000H`）
- **逻辑地址 = 段基地址 : 偏移地址**（`e.g. 3000H:2000H`）
	- 段基地址保存地址的**高16位**
	- 偏移地址也为**16位**
	- **物理地址 = 段基地址 x 16 + 偏移地址**

## 计算机主机与外设传送数据的方式：
1. 通过处理器执行 I/O 指令完成：

	- **查询传送**：需要进行数据交换时，先查询外设的工作状态，外设准备好后才可以进行数据的输入/输出。（可靠、实用性广、但查询需大量时间，效率低）
	- **中断传送**
2. 使用硬件直接处理：
	- 直接存储器存取（DMA）
	- 使用专门的 I/O 处理机
## ✅查询传送：
- 一般为获取工作状态，会设计专门查询状态的电路，状态保存在对应的**状态寄存器**中，通过**状态端口**进行读取
- 状态一般为状态**位**，一个寄存器会存储多个状态位
- 代码示例：
``` asm
// 查询输入接口（从外设获取数据）
	mov dx, 5001h   ;DX指向状态端口
status:
	in al, dx       ;读状态端口
	test al, 01h    ;测试状态位D0
	jz status
	
	; D0 = 0，设备未就绪，继续轮询
	dec dx
	
	; D0 = 1，设备就绪，DX指向数据端口
	mov dx, ...     ; DX指向数据端口
	in al, dx       ; 从数据端口输入数据到CPU
	
// 查询输出接口（输出数据给外设）
	mov dx, 5001h   ;DX指向状态端口
status:
	in al, dx
	test al, 80h    ; 测试标志位D7
	jnz status      ; D7 = 0：未就绪
	                ; D7 = 1：已就绪，转移到数据端口
	
	dec dx

	mov dx, ...
	mov al, buf     ; 将变量buf的值送到al中
	; 注意：MASM中要送地址的话应该是mov ax, offset buf或者lea ax, buf
	; offset 返回的是偏移地址，系统多少位，偏移地址就是多少位
	out dx, al

```

## ✅**中断传送**：
### 基本概念：
- 中断是微机系统中非常重要的一种技术
- 利用**外部中断**，微机系统可以实时响应外部设备的数据传送请求、能够及时处理外部意外或紧急事件
- 利用**内部中断**，处理器为用户提供了发现、调试并解决程序执行时异常情况的有效途径
- 中断：处理器在执行程序过程中，被内部或外部的事件所==打断==，==转去==执行一段预先安排好的**中断服务程序**；服务结束后，又==返回==原来的断点，**==继续执行==原来的程序**
- **中断源**：引起中断的事件或原因
- **中断分类**：
	- **外部中断（硬件中断）**：
		- **可屏蔽中断**：
			- 外部通过**可屏蔽中断 INTR 请求信号**提出的中断
			- 允许可屏蔽中断（IF = 1）的条件下、当前指令执行结束予以响应
			- **输出可屏蔽中断响应信号 $\overline{INTA}$** ，产生可屏蔽中断响应总线周期，读取中断向量号
			- 需要**中断控制器**负责处理**中断优先权排队**等管理工作
			- 可屏蔽中断主要用于**与外设进行数据交换**
		- **非屏蔽中断**：
			- 外部通过**非屏蔽中断 NMI 请求信号**提出的中断
			- 处理器在**当前指令执行结束**予以响应
			- 非屏蔽中断的中断向量号是**2**
			- 非屏蔽中断主要用于处理**系统的意外或故障**
	- **内部中断（软件中断）**：由于处理器内部执行程序出现异常引起的程序中断（异常）
		- **除法错异常（向量号0）**
		- 调试异常（向量号1）
		- 断点异常（向量号3）
		- **溢出异常（向量号4）**
		- 无效代码异常（向量号6）
		- 通用保护异常（向量号13）
		- 页面失效异常（向量号14）

### ✅中断工作流程（可屏蔽中断 INTR）：
1. **中断请求**：
	- 外设向CPU发送中断请求，CPU在每条指令的最后一个时钟周期进行请求获取。
2. **中断响应**（每条指令结束后检测中断）
	- **中断响应周期**：
		- **关中断**：置IF = 0（硬件自动清零）
		- **断点保护**：按 FLAGS->CS->IP 的顺序入栈，记住中断的位置，用于之后的中断返回
		- **中断源识别**：确定发出中断的设备，并根据中断向量查询中断向量表，获取中断服务程序的段选择器CS和指令指针IP。
			- CPU生成中断响应总线周期，中断接口电路将中断向量号发送至CPU
			- 或是保存在中断状态寄存器中，处理器在获取到中断请求后**依次查询中断状态寄存器**，发现某个中断请求状态有效说明其提出了请求，并自动转向对应的中断服务程序，称为**中断源的查询识别方式**
	- **中断服务程序**：
		- **现场保护**：将通用寄存器的值入栈
		- **中断服务**：执行具体的中断处理逻辑
		- **恢复现场**：恢复之前保存的寄存器值
		- **开中断**：置 IF = 1（`STI`指令）
3. **中断返回**：弹出IP、CS、FLAGS，返回到主程序原来的位置继续执行。
### 中断请求条件：
- 是否有 INTR 信号
- 是否允许中断（IF = 1）
- 是否为非屏蔽中断（NMI）
### 中断响应条件：
- 中断类型有效（**中断类型号是一个合法的、在 0～255 范围内的 8 位无符号整数，并且对应的中断向量在中断向量表（IVT）中有定义（或至少可访问）**）
- IF=1（允许中断）
- 当前指令已完全执行
- 没有更高级的请求发生

### 中断优先权排队：
- 同时出现多个中断请求时，需要根据每个中断源被分配的**中断优先权**决定响应的先后顺序。
- **中断优先权排队**指的是系统设计者事先为每个中断源所确定的优先处理顺序
- **查询方法**实现中断优先权排队：
	- 依次查询，先查询到的先执行。查询顺序决定了优先级顺序
	- 花费时间，适用于小型微机系统
- **硬件电路**实现中断优先权排队：
	- 通过硬件电路对中断进行编码，通过硬件电路比较编号，实现中断优先权排队
	- 响应快，适用于复杂微机系统，但电路复杂。
### 中断嵌套：
- 中断嵌套（Interrupt Nesting）是指 **在执行一个中断服务程序（ISR）的过程中，CPU 响应另一个更高优先级的中断请求，并转去处理新的中断**，形成“中断中又发生中断”的嵌套结构。
- 在 x86 实模式下，默认情况下 **中断嵌套是被禁止的**，因为：
	- CPU 在响应中断时会 **自动清 IF = 0**（关中断）
	- 所以即使有新的中断请求，只要 IF=0，**可屏蔽中断（INTR）就不会被响应**
- 要实现中断嵌套，必须：
	1. **在低优先级的 ISR 中显式执行 `STI` 指令**（置 IF = 1），重新开中断
	2. 满足1之后,新来的中断**优先级比当前中断更高**
> 💡 注意：非屏蔽中断（NMI）和异常（如除零）**不受 IF 影响**，即使在 ISR 中也能嵌套发生。

### 中断描述符表和中断向量表：
- **保护方式**下，使用**中断描述符表 IDT**：
	- **中断描述符**指向中断服务程序；
	- 每个中断描述符包含**8个字节**，保存：
		- 16位段选择器和32位偏移地址
		- 中断特权层
- **实地址**方式下，使用**中断向量表**：
	- 物理地址为00000H，对应向量号从0开始；
	- 依次每**4个字节**安排一个中断向量：
		- 含有**16位段地址**和**16位偏移地址**
		- 低字部分是偏移地址、高字部分是段地址（段:偏移）
	- **256个中断**占用**1KB**区域，形成**中断向量表**
### ✅内部中断服务程序：
``` asm
// 演示如何动态修改和恢复中断向量表（IVT）
; ============================== 
; 数据段 
; ============================== 
	intoff word ? ; 用于保存原 INT 80h 的偏移地址（IP） 
	intseg word ? ; 用于保存原 INT 80h 的段地址（CS） 
	intmsg byte 'A Instruction Interrupt!', 0dh, 0ah, 0 ; 要显示的消息，以回车换行和空字符结尾 

; ============================== 
; 代码段 
; ============================== 

	; ───────────────────────────── 
	; 步骤1：获取当前 INT 80h 的中断向量（保存原 ISR 地址） 
	; 35h: 获取中断向量表的项目
	; 输入参数：AL = 中断类型号 (80h) 
	; 返回值: ES:BX = 原中断向量 (ES=段, BX=偏移) 
	; ───────────────────────────── 
	mov ax, 3580h ; AH=35h (功能号), AL=80h (中断号) 
	int 21h ; 调用 DOS 
	mov intoff, bx ; 保存原 ISR 的偏移地址 
	mov intseg, es ; 保存原 ISR 的段地址 
	
	; ───────────────────────────── 
	; 步骤2：设置新的中断向量为 new80h 
	; 25h: 设置中断向量表的项目
	; 输入参数：AL = 中断类型号 (80h) 
	; DS:DX = 新 ISR 的地址 
	; ───────────────────────────── 
	push ds ; 保存当前 DS（因为要修改 DS 用于设置向量） 
	
	mov dx, offset new80h ; DX = new80h 的偏移地址 
	mov ax, seg new80h ; AX = new80h 所在段（通常是 CS 或 DS） 
	mov ds, ax ; DS:DX 指向 new80h 
	
	mov ax, 2580h ; AH=25h (设置向量), AL=80h (中断号) 
	int 21h ; 调用 DOS，安装新 ISR 到 INT 80h 
	
	pop ds ; 恢复原来的 DS 
	; ───────────────────────────── 
	; 步骤3：触发 INT 80h，执行自定义 ISR 
	; ───────────────────────────── 
	mov dx, offset intmsg ; 将消息地址传给 DX（供 ISR 使用） 
	int 80h ; 触发中断！跳转到 new80h 
	
	; ───────────────────────────── 
	; 步骤4：恢复原来的 INT 80h 中断向量 
	; ───────────────────────────── 
	mov dx, intoff ; DX = 原 ISR 偏移 
	mov ax, intseg ; AX = 原 ISR 段 
	mov ds, ax ; DS:DX = 原 ISR 地址 
	
	mov ax, 2580h ; AH=25h, AL=80h 
	int 21h ; 恢复原中断向量 
	
	; （程序结束，可加 exit 调用） 
	; mov ax, 4c00h 
	; int 21h 

; ============================== 
; 自定义中断服务程序：INT 80h 的新处理函数 
; 功能：从 DX 指向的字符串逐字符输出到屏幕（BIOS Teletype） 
; ============================== 
new80h proc 
	sti ; 开中断（允许嵌套，虽非必需但良好实践） 
	push ax 
	push bx 
	push si ; 保存寄存器现场 
	
	mov si, dx ; SI 指向传入的字符串（DX 由主程序设置） 
new1: 
	mov al, [si] ; 取当前字符 
	cmp al, 0 ; 是否为字符串结束符 '\0'？ 
	jz new2 ; 是 → 跳转到结束 
	mov ah, 0eh ; BIOS 功能：Teletype Output 
	mov bx, 0 ; 页面号=0，前景色（文本模式下常忽略） 
	int 10h ; 调用 BIOS 显示字符 
	inc si ; 指向下一个字符 
	jmp new1 ; 继续循环 
	
new2: 
	pop si 
	pop bx 
	pop ax ; 恢复寄存器现场 
	iret ; 中断返回（自动恢复 FLAGS, CS, IP） 
new80h endp
```

## ✅中断控制器8259A：
### 概念：
- 管理多个中断请求并进行优先权排队等工作 
- IBM PC/AT机使用**两个Intel 8259A可编程中断控制器**PIC，32位PC机兼容了它们的功能 
	- 每个管理8级中断，请求引脚：IR0～IR7 
	- 每一级中断可单独被屏蔽或允许 
	- 多个芯片级联最多扩展至64级中断 
	- 为每级中断提供中断向量号

### 中断寄存器：
- **中断请求寄存器IRR**：
	- 保存8条外界中断请求信号 IR0～IR7 的请求状态（记录**当前有哪些中断请求到来**（但尚未被响应））
	- Di 位为 1 表示 IRi 引脚有中断请求
- **中断服务寄存器ISR**：
	- 保存正在被8259A服务的中断状态（记录**当前正在被 CPU 处理的中断**（即已响应但未结束））
	- Di 位为 1 表示 IRi 中断正在服务中
- **中断屏蔽寄存器IMR**：
	- 保存对中断请求信号IR的屏蔽状态（记录**哪些中断被屏蔽**）
	- Di 位为 1 表示 IRi 中断被屏蔽（禁止）

### 中断处理方式：
#### 1. 普通全嵌套方式（Fully Nested Mode）：
> 这是 8259A **默认的中断优先级管理方式**。
##### 🔹 核心规则：
1. **中断优先级固定**：  
    IR0 > IR1 > IR2 > … > IR7（IR0 优先级最高）。
2. **CPU 总是响应当前所有请求中优先级最高的那个中断**。
3. **一旦开始处理某个中断 IRi**：
    - 8259A 会将 ISR 的第 i 位置为 1（表示“IRi 正在服务中”）；
    - **在此期间，禁止响应任何优先级 ≤ IRi 的中断**（即同级和低级）；
    - **但允许优先级更高的中断（IR0 ～ IRi−1）打断当前 ISR，实现嵌套**。
##### 🔹 举个例子：
- 当前正在处理 **IR3**（ISR\[3\] = 1）
- 此时：
    - 若 IR4、IR5、IR6、IR7 请求 → **被屏蔽**（优先级更低）
    - 若 IR2、IR1、IR0 请求 → **可以被响应**（优先级更高），形成中断嵌套

> 💡 这就是“全嵌套”的含义：**高优先级中断可以嵌套打断低优先级中断**。

#### 2. 普通中断结束方式（Normal End of Interrupt, Normal EOI）
> 当中断服务程序（ISR）执行完毕后，**必须通知 8259A：“我处理完了！”**，否则 8259A 会认为该中断仍在处理，继续屏蔽同级/低级中断。
##### 🔹 运行方式：
- 中断开始时，8259A 自动置对应 ISR 位为1；中断结束前，必须手动向 8259A 发送 EOI 命令，通知 8259A ISR 已执行结束，8259A 会清除 ISR 中**当前优先级最高的置1位**；若不发 EOI，该中断将持续屏蔽同级/低级请求。
``` asm
mov al, 20h    ; EOI 命令（二进制 0010 0000）
out 20h, al    ; 发送给主 8259A，让其将IR2清零
```
##### 🔹 8259A 如何处理 EOI？
- 在 **普通 EOI 模式** 下，8259A **自动将 ISR 中优先级最高的那一位清零**（即最后被响应的那个中断）。
- 因为在全嵌套模式下，**ISR 中置 1 的位一定是按优先级从高到低排列的**，所以“最高优先级的 ISR 位”就是**最近一次被响应的中断**。
##### 🔹 为什么这样设计？
- 为了支持**中断嵌套**：  
    - 例如先响应 IR3，再被 IR1 打断 → ISR = \[IR1=1, IR3=1\]  
    - 执行 EOI → 清除 IR1（最高优先级）→ 继续处理 IR3  
    - 再执行 EOI → 清除 IR3 → 中断全部结束
> ✅ 这种机制天然匹配“后进先出（LIFO）”的中断嵌套返回顺序。

### 8259A编程：
- 目的：指定其**工作方式**和**控制中断处理过程**。
- 主片：
	- I/O地址：**20H(ICW1，OCW2~OCW3)、21H(ICW2~ICW4，OCW1)**
	- 对应 IRQ0~IRQ7 的中断向量号为 **08H~0FH**
- 从片：
	- I/O地址：**A0H(ICW)、A1H(OCW)**
- 开始工作前，必须进行**初始化编程**：
	- 写入初始化命令 ICW：选择工作方式（系统开机时已自动由BIOS初始化，应用程序**不应该对其初始化**，否则会破坏系统中断）
- 工作期间，需要**中断操作编程**：
	- 写入**操作命令字 OCW**：按新要求工作，读取信息
	- OCW1：写入IMR，控制中断屏蔽（写入21H）
	- OCW2：产生EOI命令，改变优先权顺序（写入20H）
	- OCW3：控制读取IRR、ISR、IMR和查询字（写入20H）

### 应用程序只需要写入 OCW，所以接下来只介绍 OCW 的写入方式
#### **OCW1：屏蔽中断**：
``` asm
mov al, 00001010b ; 屏蔽 IR1 和 IR3（注意：D1 和 D3 = 1） 
out 21h, al ; 写入 OCW1（即写 IMR）
```

------
#### **OCW2：改变优先级/发送EOI**：

| D7                           | D6                                                                                                 | D5                         | D4  | D3  | D2                    | D1  | D0  |
| ---------------------------- | -------------------------------------------------------------------------------------------------- | -------------------------- | --- | --- | --------------------- | --- | --- |
| R（R=1为旋转优先级控制模式，本书没讲，所以默认填0） | SL（**是否指定中断** ：**SL=0**时为**非指定**，8259A自动处理结束哪个中断；**SL=1**时为**指定中断**，由 D2~D0 ,即 L2~L0 字段明确指定结束哪个中断） | EOI（EOI=1，表明该命令为中断结束命令EOI） | 0   | 0   | L2（与D1和D0搭配使用，指定IR引脚） | L1  | L0  |
 - **普通 EOI（结束最高优先级中断）**
``` asm
mov al, 20h    ; 0010 0000 → R=0, SL=0, EOI=1
out 20h, al
```
> → 清除 ISR 中**优先级最高的置1位**

- **指定 EOI（结束特定中断）**
``` asm
mov al, 65h    ; 0110 0101 → R=0, SL=1, EOI=1, L2-L0=101 (IR5)
out 20h, al
```
> → 清除 ISR\[5\]（用于非全嵌套或特殊场景）

#### **OCW3：设置 8259A 的“读取模式”，决定接下来从端口 `20h` 读取的是中断请求寄存器 IRR 还是中断服务寄存器 ISR。**

| D7  | D6  | D5  | D4  | D3   | D2  | D1  | D0  |
| --- | --- | --- | --- | ---- | --- | --- | --- |
| 0   | 0   | 0   | 0   | ESMM | SMM | 1   | 1   |
- 常用位如下：

| 位     | 名称                        | 常用值   | 作用                      |
| ----- | ------------------------- | ----- | ----------------------- |
| D1,D0 | 必须为 `11`                  | 固定    | 表示这是 OCW3（不是 OCW2）      |
| D2    | RIS（Read IRR/ISR Select）  | 0 或 1 | 0 = 下次读 ISR，1 = 下次读 IRR |
| D3    | RR（Read Register Command） | 1     | 表示“我要读寄存器”              |

>✅ 最常用的两个 OCW3 值：
>	- **`0AH`（0000 1010）** → 读 **IRR**
>	- **`0BH`（0000 1011）** → 读 **ISR**
``` asm
; 读 IRR（当前请求） 
mov al, 0Ah ; 0000 1010 → RIS=1, RR=1, P=0 
out 20h, al ; 向20h中写入OCW3
in al, 20h ; 从20h中读取IRR，AL = IRR 

; 读 ISR（当前服务中） 
mov al, 0Bh ; 0000 1011 → RIS=1, RR=1, P=1 
out 20h, al ; 向20h中写入OCW3
in al, 20h ; 从20h中读取ISR，AL = ISR

// 注意：读取位置是20h！！！！！！！！！！！！！！
```
#### ✅程序示例：
``` asm
// 安装一个自定义的 INT 08h 中断服务程序（ISR），用于响应 PC 的定时器中断（IRQ0）。  
// 每次中断触发时，显示一条消息并计数；
// 当计数达到 10 次后，主程序退出，并恢复原始中断向量和中断屏蔽状态。

	; 数据段
	intmsg byte 'A 8259A Interrupt !', 0dh, 0ah, 0 ; 要显示的字符串（带回车换行）
	counter byte 0 ; 中断触发次数计数器（初始为0）

	; 代码段
	
	; ───────────────────────────── 
	; 步骤1：保存原 INT 08h 的中断向量 
	; DOS 功能 35h: Get Interrupt Vector 
	; AL = 08h → 获取 INT 08h 的向量 
	; 返回: ES:BX = 原 ISR 地址 
	; ───────────────────────────── 
	mov ax, 3508h ; AH=35h (功能号), AL=08h (中断号) 
	int 21h ; 调用 DOS 
	push es ; 保存原 ISR 段地址 
	push bx ; 保存原 ISR 偏移地址
	
	; ───────────────────────────── 
	; 步骤2：安装新的 INT 08h ISR (new08h) 
	; DOS 功能 25h: 设置中断向量表项 
	; AL = 08h ; DS:DX = 新 ISR 地址 
	; ───────────────────────────── 
	cli ; 关中断，防止在修改向量时被中断打断 
	push ds ; 保存当前 DS 
	
	mov ax, seg new08h     ; AX = new08h 所在段（通常是 CS 或 DS） 
	mov ds, ax             ; 设置 DS 指向 new08h 所在段 
	mov dx, offset new08h  ; DX = new08h 的偏移地址 
	mov ax, 2508h          ; AH=25h (设置向量), AL=08h 
	int 21h                ; 安装新 ISR 到 INT 08h 
	
	pop ds ; 恢复原来的 DS
	
	; ───────────────────────────── 
	; 步骤3：开放 IRQ0 中断（IR0） 
	; 8259A 的 IMR 寄存器位于端口 21h 
	; 位0 = IR0 屏蔽位（1=屏蔽，0=允许） 
	; ───────────────────────────── 
	in al, 21h ; 读取当前 IMR（中断屏蔽寄存器） 
	push ax ; 保存原始 IMR 
	and al, 0FEh ; 清除 bit0（0FEh = 1111 1110b），允许 IRQ0 
	out 21h, al ; 写回 IMR，启用定时器中断 
	
	; ───────────────────────────── 
	; 初始化计数器并开中断，开始等待 
	; ───────────────────────────── 
	mov counter, 0 ; 中断次数清零 
	sti ; 开中断，允许 IRQ0 触发 INT 08h

; ───────────────────────────── 
; 主循环：等待中断发生 10 次 
; ───────────────────────────── 
start1: 
	cmp counter, 10 ; 比较中断次数是否达到 10 
	jb start1 ; 若 <10，继续循环等待 
	; ───────────────────────────── 
	; 步骤4：恢复系统状态 
	; ───────────────────────────── 
	cli ; 先关中断，确保恢复过程不被打断 
	
	pop ax ; 弹出原始 IMR 
	out 21h, al ; 恢复 IMR（重新屏蔽 IRQ0，如果原来被屏蔽） 
	
	pop dx ; 弹出原 ISR 偏移 
	pop ds ; 弹出原 ISR 段（注意：DOS 25h 要求 DS:DX） 
	mov ax, 2508h ; AH=25h, AL=08h 
	int 21h ; 恢复原始 INT 08h 向量 
	
	sti ; 开中断（可选，程序即将结束）
	
; ============================== 
; 自定义 INT 08h 中断服务程序 
; 功能：显示消息、计数、发送 EOI 
; ============================== 
new08h proc 
	sti ; 开中断（允许高优先级中断嵌套） 
	; 保存现场（C语言调用约定要求保存这些寄存器） 
	push ax 
	push si 
	push ds 
	; 设置 DS 指向数据段（因为 ISR 可能从任意代码段调用） 
	mov ax, @data ; @data 是 MASM/TASM 的伪操作符，表示数据段地址 
	mov ds, ax ; 更新中断计数 
	
	inc counter ; 中断次数 +1 ; 显示提示信息（假设 dpstri 是一个输出字符串的子程序） 
	mov si, offset intmsg 
	call dpstri ; 调用字符串输出函数（需自行实现） 
	
	; 发送 EOI（End of Interrupt）给 8259A 
	mov al, 20h ; 普通 EOI 命令（清除 ISR 中最高优先级位） 
	out 20h, al ; 写入 8259A 命令端口 
	
	; 恢复现场 
	pop ds 
	pop si 
	pop ax 
	
	iret ; 中断返回（恢复 FLAGS, CS, IP） 
new08h endp
```

## 驻留中断服务程序：
- 没有驻留的程序执行结束后，使用的主存空间被DOS**回收**用于其他程序
- **驻留TSR程序**：程序执行结束保存在主存，程序结束后代码仍存在于内存中，还能被其他程序调用。
- 中断服务程序要让其他程序使用**必须驻留**
- 用DOS功能调用的**31H**号实现程序驻留并返回
	- AH = 31H
	- 入口参数：AL = 返回代码；DX = 驻留长度（单位：**节**；1节 = 16字节）
#### 代码示例：
``` asm
include io16.inc 

.code 
;---------------------------------------------------------- 
; 主程序：安装中断向量 + 显示信息 + 驻留内存 
;---------------------------------------------------------- 
start: 
	mov ax, cs 
	mov ds, ax ; 设置 DS = CS，以便访问本段中的数据（如 tsrmsg、new04h） 
	
	; 安装新的 INT 04h 中断向量 
	mov dx, offset new04h ; DX = 新 ISR 的偏移地址（段地址已在 DS 中 = CS） 
	cli ; 关中断，防止在修改中断向量表时被干扰 
	mov ax, 2504h ; AH = 25h（DOS “设置中断向量” 功能），AL = 04h（INT 04h） 
	int 21h ; 调用 DOS，将 (DS:DX) 写入中断向量表第 4 项（地址 0000:0010h） 
	sti ; 开中断 ; 显示 TSR 安装成功的信息 
	
	mov ax, offset tsrmsg ; AX = 提示字符串的偏移地址 
	call dispmsg ; 调用子程序 dispmsg（由 io16.inc 提供，显示字符串） 
	
	; 计算需要驻留的内存大小（单位：paragraphs，即 16 字节/节） 
	mov dx, offset program_end ; DX = 程序尾部
	sub dx, offset start       ; 计算程序长度
	add dx, 15 ; 为向上取整做准备（+15 后除以 16 相当于 ceil(n/16)） 
	shr dx, 4 ; DX = (offset start + 256 + 15) / 16 → 转换为节（paragraphs） 
	
	; 调用 DOS 功能 31h：驻留并退出（TSR 的关键！） 
	mov ax, 3100h ; AH = 31h（Keep Process），AL = 00h（退出码） 
	int 21h ; 程序终止，但保留 DX 指定的段数在内存中 
	
; 安装成功提示信息（以 0 结尾，供 dispmsg 使用） 
tsrmsg byte 'INT 04H Program Installed !', 0dh, 0ah, 0 
; 要显示的溢出提示信息（以 0 结尾，用于自定义字符串输出） 
intmsg byte 0dh, 0ah, 'Overflow !', 0 

;---------------------------------------------------------- 
; 自定义的 INT 04h 中断服务程序（处理溢出异常） 
;---------------------------------------------------------- 
new04h proc ; 定义中断服务程序 new04h 
	sti ; 开中断（允许更高优先级中断嵌套） 
	
	; 保存被中断程序的寄存器现场（顺序很重要） 
	push ax 
	push bx 
	push si 
	push ds 
	
	; 设置 DS = CS，因为字符串 intmsg 定义在代码段中 
	mov ax, cs 
	mov ds, ax 
	mov si, offset intmsg ; SI 指向要显示的字符串 "Overflow !" 
	
dps1: 
	mov al, [si] ; 从内存读取一个字符到 AL 
	cmp al, 0 ; 判断是否为字符串结束符（0） 
	jz dps2 ; 若是，则跳转到恢复现场 
	
	; 调用 BIOS 视频服务：AH=0Eh 表示“电传打字方式输出字符” 
	mov bx, 0 ; BH = 页面号（通常为 0），BL 在 AH=0Eh 时被忽略 
	mov ah, 0eh 
	int 10h ; 调用视频 BIOS 显示 AL 中的字符 
	
	inc si ; 指向下一个字符 
	jmp dps1 ; 循环继续 
dps2: 
	; 恢复寄存器现场（顺序与压栈相反） 
	pop ds 
	pop si 
	pop bx 
	pop ax 
	
	iret ; 中断返回（恢复标志、CS、IP） 
new04h endp ; new04h 过程结束 

program_end:  ; 整个程序的末尾

    ; 程序入口点声明 
	end start
```

## DMA传送：
- **DMA（Direct Memory Access，直接内存访问）** 是一种允许外设（如磁盘、网卡、显卡等）在**不经过 CPU 干预**的情况下，**直接**与系统**内存**进行数据交换的技术。
- **核心思想**：让 CPU 从繁重的数据搬运工作中解放出来，提高系统整体效率。
### 工作过程简述：
1. **DMA预处理**：CPU 设置 DMA 控制器，告诉它：
	- 工作方式
    - 源地址（例如外设的 I/O 缓冲区）
    - 目标地址（内存中的缓冲区）
    - 要传输的数据长度
    - 等等等等......
2. **启动传输**：
	- 外设向DMA控制器发送**DMA请求（DMAREQ）** 信号；
	- DMA控制器收到 DMAREQ 信号后，向CPU发送**总线请求（HOLD）** 信号，请求接管总线（该信号在整个传输过程中**一直有效**）；
	- CPU在当前总线周期结束后响应该请求，并返回**总线响应（HLDA）** 信号，表示已放弃总线控制;
	- DMA控制器接收到 HLDA 信号后，返回**DMA响应（DMAACK）** 信号，开始DMA传送
3. **DMA 数据交换**：DMA 控制器接管总线，直接在外设和内存之间搬移数据。
	- DMA**读**：存储器 -> 外设（**以外设为中心**）
	- DMA**写**：外设 -> 存储器
	- 都得通过数据总线传输数据，只不过不用CPU控制。
4. **判断传送是否完成**：
	- DMA控制器会**对存储器地址进行增/减量**，并**对传送次数进行计数**，据此判断数据块传送是否完成：
		- 若未完成，一直重复3~4；
		- 若完成，DMA控制器将HOLD信号转为**无效**，表示传送结束，将总线控制权**交还**给CPU
### DMA控制器8237A：
- 8237A 每个DMA通道有**4种**工作方式：
	- **单字节传送方式**：每次DMA传送时仅传送一个字节
	- **数据块传送方式**：连续地传送数据，直到规定的字节数传送完
	- **请求传送方式**：
		- 请求有效，连续传送数据
		- 请求无效，DMA传送被暂时中止
		- 再次有效，DMA传送又继续
	- **级连方式**：多个DMA控制器连接起来扩展DMA通道

## 简答题：
1. **为什么称8253/8254的工作方式1为可编程单稳脉冲工作方式？**
    - 方式1可以通过编程产生一个确定宽度的单稳脉冲，故称工作方式1为可编程单稳脉冲工作方式。
2. **为什么写入8253/8254的计数初值为0却代表最大的计数值？**
    - 因为计数器是先减1，再判断是否为0，所以写入0实际代表最大计数值（即65536，对应16位计数器）。
3. **处理器通过8255的控制端口可以写入方式控制字和位控制字，8255如何区别这两个控制字呢？**
    - 通过控制字的D7位来区别：D7＝1，该控制字为方式控制字；否则为位控制字。
4. **“8255具有锁存输出数据的能力”是什么意思？**
    - 8255的三种工作方式均可实现输出数据锁存，即数据输出后被保存在8255内部，可以读取出来，只有当8255再输出新一组数据时才改变。
5. **Modem（戏称“猫”）是一个什么作用的器件？**
    - Modem，称为调制解调器，将数字信号转换为适合在电话线路上传送的模拟信号（调制）以及将电话线路的模拟信号转换为数字信号（解调）。
6. **RS-232C标准使用25针连接器，为什么PC机上常见的是9针连接器？**
    - 因绝大多数设备只使用RS-232C标准的其中9个信号，所以PC机上就配置9针连接器。
7. **什么是RS-232C的零调制解调器连接方式？**
    - 两台微机进行短距离通信，可以不使用调制解调器，直接利用232C接口连接，被称为零调制解调器（Null Modem）连接。
8. **UART器件的主要功能是什么？**
    - UART表示通用异步接收发送器，主要功能是将并行数据转换为串行数据发送，以及实现串行数据转换为并行传送给处理器。
9. **多路开关在模拟输入输出系统中起什么作用？**
    - 采用多路开关，通过微型机控制，把多个现场信号分时地接通到A/D转换器上转换，达到共用A/D转换器以节省硬件的目的。
10. **处理器为什么需要通过锁存器与数字/模拟转换器连接？**
    - 处理器输出数据都只在输出指令OUT执行的极短时间内出现在数据总线上，慢速的外设不能及时获取，所以主机与DAC之间必须连接数据锁存器。

## 判断题:
1. **称为定时器也好，称为计数器也好，其实它们都是采用计数电路实现的。**
    - ✅
    - 定时本质上是对**固定频率时钟脉冲**的计数，因此定时器和计数器在硬件结构上是一致的。
2. **32位PC机中并没有8253或8254芯片，但其控制芯片组具有兼容其功能的电路。**
    - ✅
    - 现代PC将8254功能集成到南桥或PCH芯片组中，保留了对传统定时器的兼容性。
3. **计数可以从0开始逐个递增达到规定的计数值，也可以从规定的计数值开始逐个递减恢复到0；前者为加法计数器，后者是减法计数器；8253/8254采用后者。**
    - ✅
    - 8253/8254是**减法计数器**，初值写入后从该值开始递减至0。
4. **一次实现16位并行数据传输需要16个数据信号线。进行32位数据的串行发送只用一个数据信号线就可以。**
    - ✅
    - 并行传输每位需**独立**信号线；串行传输按位依次发送，仅需**一根**数据线。
5. **8255没有时钟信号，其工作方式1的数据传输采用异步时序。**
    - ✅
    - 8255依靠**外部握手信号**（如STB#、ACK#）协调传输，属于**异步联络**方式，无需内部时钟。
6. **调制解调器的信号调制是数字信号与模拟信号的转换，所以其转换原理与ADC或DAC器件一样。**
    - ❌
    - Modem的调制/解调涉及**频谱搬移**（如FSK、QAM），远比ADC/DAC的幅度量化复杂，原理不同。
7. **模拟地线和数字地线都是地线，所以一般可以随意连接在一起。**
    - ❌
    - 模拟地和数字地应**单点连接**，避免数字噪声耦合到敏感的模拟电路，造成干扰或精度下降。
8. **电脑通过麦克风录音，需要ADC器件将音波转换为数字音频信号。**
    - ✅
    - 麦克风输出模拟音频信号，必须经**ADC采样量化**后才能被计算机处理。
9. **模拟量转换为数字量一定会引入转换误差，所以一定有失真。**
    - ✅
    - 由于量化阶跃的存在，ADC必然存在±½ LSB的**量化误差**，导致信号失真。
10. **当处理器提供数字量后DAC器件将输出相应的模拟量，但ADC器件需要启动转换，隔一定时间后才能获得数字结果。**
    - ✅
    - DAC是**即时转换**；ADC需**启动**（如写控制字）并**等待转换完成**（通过查询或中断），具有延迟。

## 定时控制接口：
- 定时控制具有极为重要的作用
	- 微机控制系统中常需要定时中断、定时检测、定时扫描等
	- 实时操作系统和多任务操作系统中要定时进行进程调度
	- PC机的日时钟计时、DRAM刷新定时和扬声器音调控制都采用了定时控制技术
- 可以使用可编程定时器芯片构建**软硬件结合**的定时电路
- 也可以让CPU执行延时子程序，采用**软件延时方法**实现定时
## 定时器（计数器）：
- 由数字电路中的计数电路构成，记录输入脉冲的个数
- 脉冲信号具有一定**随机性**，通过脉冲的个数可以获知外设的状态变化次数（**计数**）
	- 人话：外设的状态变化产生脉冲信号，**统计脉冲信号的个数**实现计数
- 脉冲信号的**周期固定**(使用高精度晶振产生脉冲信号)，个数乘以周期就是时间间隔（**定时**）
	- 人话：采用晶振产生稳定的脉冲信号，通过 `信号个数 x 周期` 可得到较为精确的时间间隔。
- **8253/8254定时器**：
	- 3个独立的16位计数器通道
	- 每个计数器有6种工作方式
## ✅8253 定时器：
- 注意：8253和8254都是**递减**计数器！！！！
- 3个定时器
- 每个定时器有3个接口：
	- **CLK（时钟输入）**：每接收一个时钟/脉冲信号（下降沿），计数器的计数值 - 1
	- **GATE（门控输入）**：控制计数器的工作，可分为**电平控制**和**上升沿控**制两种类型。
	- **OUT（计数器输出）**：当一次计数过程结束（计数值减为0）时，OUT脚产生一个**输出信号**，具体信号长什么样取决于计数器设置的工作方式
- 其引出8根**数据引脚**D7~D0，用于**接收**处理器的控制字和计数值，以及**发送**计数器的当前计数值和工作状态。
- 读写控制逻辑拥有读信号$\overline{RD}$、写信号$\overline{WR}$，地址信号$A_0$、$A_1$，以及片选信号$\overline{CS}$，工作模式如下：

| --------------- | ----- | ----- | ------------------ | ------------------ | -------- |
| 0               | 0     | 0     | 读计数器0              | 写计数器0              | 40H      |
| 0               | 0     | 1     | 读计数器1              | 写计数器1              | 41H      |
| 0               | 1     | 0     | 读计数器2              | 写计数器2              | 42H      |
| 0               | 1     | 1     | 无操作                | 写控制字               | 43H      |
| 1               | -     | -     | 无操作                | 无操作                |          |

### 控制字：
- 8253有6种工作方式，由方式控制字决定
- 方式控制字需要在**使用8253前**写入其**控制寄存器**
- 控制字如下：

| D7 D6 | D5 D4 | D3 D2 D1 | D0      |
| ----- | ----- | -------- | ------- |
| 计数器选择 | 读写格式  | 工作模式     | BCD/二进制 |
- **计数器选择**：
	- **00**：计数器0；**01**：计数器1；**10**：计数器2；**11**：在8253非法，在8254是读回命令
- **读写格式**：
	- **00**：计数器锁存命令，用于读取当前计数值
	- **01**：只读写低8位
	- **10**：只读写高8位
	- **11**：先读写低8位，再读写高8位（16位计数）
- **工作模式**：
	- **000**：方式0；**001**：方式1；......；**101**：方式5
	- 方式2和方式3的 D3 不做限制
- **D0**：
	- **0**：**二进制**计数
	- **1**：**BCD码十进制**计数

### 工作方式：
- 6种方式的工作流程大致相同，如下所述：
	1. 处理器写入方式控制字，设定**工作方式**；
	2. 处理器写入预置寄存器，设定**计数初值**；
	3. 【特殊】对方式1和方式5，需要硬件启动，即GATE端需出现一个上升沿信号；其他方式不用；
	4. 将预置寄存器内的计数初值送入计数器
	5. CLK端每接收一个下降沿信号，计数器值 - 1。计数过程受GATE信号控制，当GATE信号为**低电平**时，不进行计数。
	6. 当计数器值 = 0时，计数过程结束，OUT端输出一个信号
- 对**方式0、1、4、5**，一次计数结束后，计数器停止计数
- 对**方式2、3**，计数结束后，会将预置寄存器的计数初值重新送入计数器，重新开始下一次计数，直到写入新的方式控制字后才停止计数。

| 模式  | 名称             | 简单描述                                                                                          | OUT 信号特点                                       |
| --- | -------------- | --------------------------------------------------------------------------------------------- | ---------------------------------------------- |
| 方式0 | **计数结束中断**     | 写入控制字后OUT拉低并保持，写入计数值后自动开始计数，结束时OUT由低变高，可作为中断请求信号                                              | 计数完 → 变高并保持，直到重新装入计数值或工作方式                     |
| 方式1 | **可编程单稳脉冲**    | 写入控制字后**OUT拉高**，GATE 上升沿启动，**OUT降低**，并保持到计数结束，**OUT重新拉高**                                     | 输出 一个长低电平脉冲（宽度 = 计数值）                          |
| 方式2 | **分频器（频率发生器）** | 自动重复计数，若计数值为N，则每N个脉冲（1个计数过程），OUT输出一个宽为1CLK的负脉冲                                                | 每 N 个 CLK 输出 1 个 CLK 宽的低脉冲                     |
| 方式3 | **方波发生器**      | 写入控制字后**OUT拉高**。计数值为奇数时，每个脉冲-2；计数值为偶数时，第一个脉冲-1，之后的脉冲-2；每个计数过程近似产生半个方波（**过程好像有问题，直接记信号特点就行了**） | **从高电平开始**，高低交替，近似 50% 占空比方波（偶数对称，奇数高电平多一个CLK） |
| 方式4 | **软件触发选通信号**   | 一次性计数，写入控制字后OUT拉高，**软件写入初值后自动开始**；可由低电平GATE停止计数，GATE拉高后，重新由初值开始计数**（与Intel文档存在差别，正常应该是恢复计数）** | 计数完 → 1 个 CLK 宽的低脉冲，之后再次拉高                     |
| 方式5 | **硬件触发选通信号**   | 与方式4基本一致，只不过启动计数需要一个高电平GATE信号                                                                 | 计数完 → 1 个 CLK 宽的低脉冲，之后再次拉高                     |
### 编程示例：
``` asm

mov al,30h   ; 方式控制字：30H＝00 11 000 0B（计数器0，16位读写，方式0，二进制计数）

mov al,ah
out 40h,al  ;写入高字节计数初值
```

### 三种工作方式（由控制字配置）

#### ✅ **方式 0：基本输入/输出（最常用）**

- 所有端口（A、B、C）都可独立设为 **输入或输出**；
- **无握手信号**，简单直连，CPU只需要使用输入输出指令就可以与外设进行数据交换
- 适用于 LED、开关、简单数据传输等。

#### ✅ **方式 1：选通输入/输出（带握手）**

- 只有 Port A 和 Port B 可作为 **带应答的输入/输出端口**；
- Port C 的部分引脚用作 **握手信号**（如 STB̅、IBF、ACK̅、OBF 等）；
- 还提供有中断请求逻辑和中断允许触发器，对输入和输出的数据进行锁存；
- 适用于 **打印机、键盘** 等需要同步的设备（适用于查询和中断方式的接口电路）。

#### ✅ **方式 2：双向总线方式（仅 Port A 支持）**

- Port A 作为 **双向数据总线**（既能输入也能输出）；
- Port C 提供完整的 **双向握手信号**；
- 适用于 **与 CRT 控制器、磁盘控制器** 等双向通信设备。

> 📌 Port B **不能工作在方式 2**；Port C **不能单独设为方式 1 或 2**，只能配合 A/B 使用。


### 方式1的信号分配：
#### 1. 当 **Port A 工作在方式1** 时：

| Port C 引脚 | 作用（输入模式）                                                                 | 作用（输出模式）                                                    |
| --------- | ------------------------------------------------------------------------ | ----------------------------------------------------------- |
| PC3       | $INTR$（中断请求，输出）                                                          | $INTR$（中断请求，输出）                                             |
| PC4       | $\overline{STB}$（**硬件引脚**，外设选通信号，输入）；$INTE$（**内部状态**，中断允许控制位，通过控制字进行位修改） | -                                                           |
| PC5       | $IBF$（输入缓冲满时有效，允许输出）                                                     | -                                                           |
| PC6       | -                                                                        | $\overline{ACK}$（**硬件引脚**，应答信号，输出）；$INTE$（**内部状态**，中断允许控制位） |
| PC7       | -                                                                        | $\overline{OBF}$（输出缓冲空时有效，允许输出）                             |
#### 2. 当 **Port B 工作在方式1** 时：

| Port C 引脚 | 作用（输入）                                    | 作用（输出）                                    |
| --------- | ----------------------------------------- | ----------------------------------------- |
| PC2       | $\overline{STB}$（选通信号，输入）；$INTE$（中断允许控制位） | $\overline{ACK}$（应答信号，输出）；$INTE$（中断允许控制位） |
| PC1       | $IBF$（输入缓冲满，输出）                           | $\overline{OBF}$（输出缓冲空，输出）                |
| PC0       | $INTR$（中断请求，输出）                           | $INTR$（中断请求，输出）                           |
> 功能一样，引脚不同而已

#### 3. 工作过程简述  

##### 🔹 **方式1 输入（如接键盘）**：（中心为CPU）
1. 外设将数据放到端口，并发送 **$\overline{STB}$（低电平有效）脉冲**；
2. 8255 将数据锁存，并**置 $IBF$ 为高电平**，表示“输入缓冲满”，告诉外设 8255 已锁存数据，并阻止其发送新数据；
3. 若允许中断（$INTE = 1$），则 $INTR = 1$，通知 CPU 读取数据；
4. CPU 响应中断，执行输入指令`IN`，并发出 $\overline{RD}$ 信号，从8255的 $D_0$~$D_7$ 数据脚读取数据，读取完后拉高$\overline{RD}$，表示读取结束；
5. 8255 在 $\overline{RD}$ 上升沿将 $IBF$ 置为低电平，通知外设可发送新数据；
##### 🔹 **方式1 输出（如接打印机）**：
1. CPU 响应中断，执行输出指令`OUT`，将数据写入 Port A，并发出 $\overline{WR}$ 信号；
2. $\overline{WR}$ 信号在下降沿锁存数据到输出锁存器，并立刻令 $\overline{OBF} = 0$ ，**通知外设可接收数据**
3. 当外设检测到 $\overline{OBF} = 0$ 时，从端口读取数据，并发回 $\overline{ACK}$ 响应 
4. 8255 收到 $\overline{ACK}$ 后，置 $\overline{OBF} = 1$（已接收缓冲区内的数据，缓冲区为空）。若$INTE_A = 1$，则自动置$INTR = 1$，向CPU请求中断，通知 CPU 可发下一字节。

``` asm
mov al, 00000111b ; 置位控制字，置PC3为1

```


### 具体应用：
- 端口A，B，C和控制字地址：
	- 60H，61H，62H和63H
- 工作在**基本输入/输出方式**（方式0）：
	- 端口A为方式0输入，用来读取键盘扫描码
	- 端口B为方式0输出，例如控制扬声器等
	- 端口C为方式0输入，读取系统状态和配置


``` asm
printc  proc  ;AH＝打印数据

	push dx
prn:   
	mov dx,0fffch  ; 读取端口C
	in al,dx       ; 查询打印机的状态
	and al,04h     ; 忙否（PC2＝BUSY＝0）？
	jnz prn        ; PC2＝1，打印机忙，跳回开头继续轮询
	
	; PC2＝0，打印机不忙，输出
	mov dx,0fff8h  ; 端口A假定为0fff8h
	mov al,ah
	out dx,al  ;将打印数据从端口A输出到打印机
	
	;从PC7送出控制低脉冲
	mov dx,0fffeh  
	mov al,00001110b  ;使PC7＝STROBE*＝0
	out dx,al
	nop  ;产生一定宽度的低电平
	nop
	
	;产生低脉冲STROBE*信号，令打印机接收数据
	mov al,00001111b  ;使PC7＝STROBE*＝1
	out dx,al  
	
	; 恢复现场
	pop dx
	pop ax
	ret

printc  endp
```

## 键盘及其接口：
- 键盘是微机系统最常使用的输入设备
- 小键盘：适用于单板机或以处理器为基础的仪器，实现数据、地址、命令及指令等输入
- 独立键盘：通过5芯电缆与PC微机主机连接
### **✅扫描法**：
- 一般会先将行或列全部拉为低电平，检查另一方是否有变为低电平的线，**粗扫**一遍，节省资源；
- 扫到了就进入**精确定位**：
	- 先使第0行接低电平，其余行为高电平，然后检查列线是否有变为**低电位**的线，从而判断第0行是否有键闭合；
	- 再将第1行接地，检测列线是否有变为低电位的线；
	- 如此往下**一行一行地扫描**，直到最后一行；
	- 扫描过程中，发现某一行有键闭合时（列线输入中有一位为0），便在扫描中途退出；
- 通过组合行线和列线识别此刻按下哪一键；（矩阵键盘）
``` asm
// 扫描法识别按键
key1: ; 粗检，先将所有行拉低，扫一遍列，看看有没有低电平有效信号
	mov al,00       ; 低电平
	mov dx,rowport  ; rowport＝行线端口地址
	out dx,al       ; 使所有行线为低电平，准备开始检测
	mov dx,colport  ; colport＝列线端口地址
	in al,dx        ; 从列端口读取列状态
	cmp al,0ffh     ; 如果所有列都是高电平（0xFF = 11111111b），说明没按键
	jz key1         ; 跳回key1重新轮询
	
	call delay      ; 有低电平线，说明有键闭合，延迟20ms消除抖动
	mov cx,8        ; 行数送CX，作为循环次数
	mov ah,0feh     ; 行值送到AH（这里假设扫描到第0行有按键按下）

; 精确定位，赋予行初值，重新一行一行地扫描，确定列值
key2:  
	mov al,ah       ; 把当前要扫描的行值送到AL（out只能用al）
	mov dx,rowport
	out dx,al       ; 输出对应行的扫描值
	mov dx,colport
	in al,dx        ; 读进当前的列值
	cmp al,0ffh     ; 判断是否所有列都是高电平？
	jnz key3        ; 不是，说明当前行有按键按下，转key3处理
	rol ah,1        ; 无，则行移位，扫描下一行（这里是循环左移）
	loop key2       
	jmp key1        ; 8行扫完了还没找到？那就回到最开始重新找

; 定位行列值表和键码表，用于查表匹配按键
key3:  
	mov si,offset table  ; SI指向键行列值表
	mov di,offset char   ; DI指向键代码表
	mov cx,64            ; CX＝键的个数
	
; 具体匹配逻辑，将行列值转换为具体的键码
key4:  
	cmp ax,[si]   ; 与按键的行列值比较（AX = 行:列）
	jz key5       ; 相同，说明查到
	inc si        ; 不相同，继续比较（行列值的表项是word，所以inc两次，+2字节）
	inc si
	inc di        ; 键码是byte，只用 inc 1次
	loop key4
	jmp key1      ; 全部不相同，可能是噪声，返回最开始重新进行寻找

; 获取键码并处理按键释放
key5:  
	mov al,[di]  ; 获取键代码送AL
	……           ; 判断按键释放，没有则等待（等待用户松开按键（防连击））
	call delay   ; 按键释放，再次消抖
	……           ; 后续处理

; 键盘的行列值表：低字节是列值、高字节是行值
table  word 0fefeh  ; 键0的行列值
       word 0fefdh  ; 键1的行列值
       ……  ; 其他键的行列值

; 键盘的键码表
char  byte ……  ; 键0的代码值
      byte ……  ; 键1的代码值
      ……  ; 其他键的代码值
```
### PC机键盘：
- 工作过程：
	- 键盘电路正常工作时**不断地扫描键盘矩阵**；
	- 有按键，则确定按键位置之后以**串行数据**形式发送扫描码给系统板上的键盘接口电路；
	- 键按下时，发送该键的**接通扫描码**（键盘上的对应位置）；
	- 键松开时，发送该键的**断开扫描码**（接通扫描码 + 80H）；
		- 若一直按住某键，则以**拍发速率**（每秒2～30次）连续发送该键的接通扫描码；
	- 从键盘接收一个串行形式字符，进行**串并转换**；
	- 产生**键盘中断IRQ1请求**，等待读取键盘数据；
	- CPU响应中断，进入09H键盘**中断服务程序**（过程看看就行，好像有误）：
		- ① **读取键盘扫描码**：用`IN AL 60H`即可；
		- ② **响应键盘**：系统使$PB_7 ＝ 1$；
		- ③ **允许键盘工作**：系统使$PB_7 ＝ 0$；
		- ④ **处理键盘数据**
		- ⑤ 给8259A**中断结束EOI命令**，中断返回

## ✅LED数码管：
- 主要部分是**7段发光管**
- 顺时针分别称为`a、b、c、d、e、f、g`段；
	- `a~f`是外圈，`g`是中间那一横
- 有的产品还附带有一个**小数点**`h/dp`;
- 通过**7个发光段的不同组合**，可显示多种字符：
	- 主要显示`0～9`
	- 也可显示`A～F`（16进制数）
	- 还可显示个别特殊字符：`－、P`
- 分两种：
	- **共阳极结构**：**共用阳极**高电平有效，**各段**低电平有效；
	- **共阴极结构**：**共用阴极**低电平有效，**各段**高电平有效；
### 单个数码管的显示：
- 硬件方法：专用的带驱动器的LED段译码器
- 软件方法：组成显示代码表，通过查表进行译码
``` asm
ledtb:  
	byte 3fh,06h,5bh,……  ;显示代码表(0,1,2,3...)

	; 实现1个LED数码管显示
	mov bx,1            ; BX←要显示的数字
	mov al,ledtb[bx]    ; 将数字换码为显示代码：AL←LEBTB[BX]
	
	mov dx,port  ; port＝数码管端口地址
	out dx,al  ; 输出显示
```

### 多个数码管的显示：
- 软件上用**扫描方法**实现数码显示
- 硬件上用**公用的驱动电路**来驱动各数码管
- 8个数码管：用2个8位输出端口控制
	- **位控制端口**：控制当前需要显示的是哪个（位）数码管（位选信号）；
		- 共阳极和共阴极的位选信号都是**低电平有效**，具体原理你别管~
	- **段控制端口**：控制所有数码管的哪些段（`a~g`和`h/dp`）需要显示，决定具体显示什么数码
		- 所有数码管相同的段引脚并联在一起，接到段控制端口的相应引脚；
- 实现稳定的数字显示：利用**位控制**令数码管依次显示，不断重复过程（刷新，人眼的暂留效应）
	- 重复**频率**越高，数字显示越**稳定**；
	- **延时显示时间**越长，显示**亮度**就越高；
- 要实现**各种显示效果**，只需**控制重复频率和延时时间**
``` asm
// 8个共阳极数码管的依次显示
; 数据段
	leddt  byte 8 dup(0)  ; 数码缓冲区
	ledtb  byte 0c0h,0f9h,0a4h,……,86h,8eh ; 数码管的段码表，按照0～9、A～F的顺序
	timer = 10             ; 延时常量，控制每位显示时间

; 主程序
	mov si,offset leddt  ; 指向数码缓冲区
	call displed  ; 调用显示子程序
	
; 子程序：显示一次数码缓冲区的8个数码
; 入口参数：DS:SI＝缓冲区首地址
displed  proc
	; 保护现场
	push ax
	push bx
	push dx
  
    xor bx,bx       ; 清空BX
    mov ah,0feh     ; 位选信号为11111110h，指向最右边的数码管（共阳极，位选低电平有效）

led1:
	; 从 leddt 取出数字的显示编码
	mov bl,[si]       ; 从 leddt 取出要显示的数字
	inc si            ; 指向下一位
	mov al,ledtb[bx]  ; 得到显示代码：AL←LEDTB[BX]
	
	; 用 BL(数字的显示编码) 作为索引，从 ledtb查出对应段码，并点亮数码管
	mov dx,segport    ; segport为段控制端口
	out dx,al         ; 送出段码，确定该点亮哪些段
	mov al,ah         ; 取出位显示代码
	mov dx,bitport    ; bitport为位控制端口
	out dx,al         ; 送出位码，点亮当前位的数码管
	
	; 延时显示，并移位
	call delay        ; 实现数码管延时显示
	rol ah,1          ; 指向下一个数码管
	cmp ah,0feh       ; 是否指向最右边数码管
	jnz led1          ; 没有，显示下一个数字，8位数码管都要显示一遍
	
	; 恢复现场并退出
	pop dx
	pop bx
	pop ax
	ret  
displed  endp

; 软件延时子程序
delay  proc  
	; 保护现场
	push bx
	push cx
	
	mov bx, timer  ; 外循环：timer次数

delay1:  
	xor cx,cx      ; 将CX置0，此时使用loop指令会让CX = 65535，满足jnz，从而实际执行65535次循环
delay2: 
	loop delay2  ; 内循环：65535次循环
	dec bx
	jnz delay1
	
	; 恢复现场并退出
	pop cx
	pop bx
	ret
delay endp
```
## 异步串行通信接口：
- **串行通信**：将数据分解成**二进制位**，用**一条信号线**，**一位一位顺序传送**的方式
- 串行通信的优势：用于通信的**线路少**，因而在远距离通信时可以极大地降低成本
- 串行通信适合于**远距离数据传送**，也常用于**速度要求不高的近距离数据传送**
- **通信协议（通信规程）**：收发双方共同遵守，解决传送速率、信息格式、位同步、字符同步、数据校验等问题
- **串行异步通信**：以**字符**（一个字节的数据单元）为单位进行传输，字符单独封装为一帧
- **串行同步通信**：以**一个数据块（帧）** 为传输单位
	- 这种封装结构都有头、尾、校验位等必要数据，所以
### 比特率：
- **单位**：bps（bits per second，比特/秒）
- **含义**：**每秒传输的有效数据比特数**。
- 反映的是**信息传输的速度**。
- 例如：1 Mbps = 每秒传输 1,000,000 个二进制位（0 或 1）。
### 波特率：
- **单位**：Baud（波特）
- **含义**：**每秒传输的符号（Symbol）个数**。
- 一个“符号”可以代表 1 位，也可以代表多位（取决于调制方式）。
- 反映的是**信号变化的速率**（即信道上的电平/相位/频率变化次数）。

### 他俩的关系：
- **比特率 = 波特率 × 每个符号所携带的比特数**

### 示例：
1. **二进制调制**：
	- 由于符号是二进制的，每个符号只有1bit，所以**在二进制调制状态下，比特率 = 波特率**。
	- 比如 RS-232 串口
	- **注意**：串口状态下的**有效数据速率低于比特率**，因为串口传输数据是按帧计算的，帧长肯定大于一帧内的数据长度，一帧数据量 / 帧长 < 1
2. **多进制调制**：
	- 这种情况下，就得看一个符号能携带多少 bit 的数据了，满足上述公式。
## 模拟接口：
- **模拟量**：**连续**变化的物理量
- **数字量**——时间和数值上都**离散**的量
- **DAC**：数/模转换器
- **ADC**：模/数转换器
### D/A转换器（DAC）
- 将**数字量**转换成为**模拟量**（电压或电流）
- DAC芯片有多种类型：
	- 按DAC的**性能**：通用、高速和高精度等转换器；
	- 按**内部结构**：不包含、包含数据寄存器；
	- 按**位数**：8位、12位、16位等；
	- 按**输出模拟信号**：电流输出型和电压输出型
- 原理：
	- 把每位代码按其权的大小转换成相应的模拟分量，将各模拟分量相加
	- 例如：$1101\text{B}＝1×2^3＋1×2^2＋0×2^1＋1×2^0＝13$
- DAC芯片相当于一个“输出设备”
- 不带锁存器的DAC，至少需要一级**锁存器**作为接口电路
- DAC 的调用就是调用 I/O 
### A/D转换器（ADC）
- 将**模拟量**（电压或电流）转换成为**数字量**
- ADC芯片主要以**模拟到数字转换技术**来区别
- 有些ADC芯片不仅具有A/D转换的基本功能，还包含内部放大器和三态输出锁存器
- 有的还包括多路开关、采样保持器等 
### A/D转换芯片：ADC0809
- 8位逐次逼近式ADC，带多路开关和三态锁存缓冲器；
- 需要提供外部时钟；
- 控制信号：$START$（启动转换）、$ALE$（锁存通道地址）、$EOC$（转换结束）、$OE$（输出使能）。
- 与8086通过**并行 I/O 接口**（如8255）连接。
![[Pasted image 20260117162535.png]]
- **输入端**：
	- $IN_0$～$IN_7$：8个**模拟电压输入端**，可通过地址线**选择**其中一路进行A/D转换
	- $ADDA$、$ADDB$、$ADDC$：3个**地址输入线**
	- $ALE$：**地址锁存允许信号**：
		- 提供一个8通道的多路开关和寻址逻辑
		- $ALE$ 的上升沿用于锁存3个地址输入的状态
	- 译码器从8个输入中**根据地址线选择**一个模拟输入进行A/D转换
- **输出端**：
	- 输出允许信号OE：高电平有效
	- 输出端口$D_1$ ~ $D_7$：将三态锁存缓冲器中存放的转换后的 **8 位二进制数字量**从 $D_1$ ~ $D_7$ 输出。
		- **范围**：$0V$ ~ $V_{ref}$（$5V$）—— 全0 ~ 全1
- **A/D转换公式：**
$$N = \frac{V_{in} - V_{ref(-)}}{V_{ref(+)} - V_{ref(-)}} \times 2^8$$
- 其中：
	- N ：A/D 转换后的**数字输出值**（无符号**整数**，记得四舍五入），对于 8 位 ADC，理论范围是 $0≤N<256$ （即 $0$ 到 $2^8−1=255$ ）。
	- $V_{in}$ ：待转换的**模拟输入电压**。
	- $V_{ref(+)}$：**正参考电压**（通常接 +5V）。
	- $V_{ref(−)}$ ：**负参考或接地参考电压**（通常为 0V）。
	- $2^8=256$ ：8 位 ADC 的**量化级数**（注意：实际最大输出为 255）。
- 这个公式本质上是**线性比例映射**：把输入电压 $V_{in}$ 在参考电压区间 $[V_{ref(−)}, V_{ref(+)})$ 内的位置，按比例“**缩放**”到 0～255 的数字范围。
- 人话：计算输入电压与参考电压下界的差值 / 参考电压区间，得到输入电压的比例因子，再乘256，转换为十进制数
### ADC芯片与主机的连接：
- ADC芯片被视为一种可编程的输入外设，相当于“**输入设备**”；
- ADC的输出**不够稳定**，不能将ADC数据线挂到数据总线，需要接口电路提供**数据缓冲器**，令CPU能够安全读取稳定的数据；
- 转换开始需要**启动信号**，一般是软件编程或硬件定时产生**脉冲信号或电平信号**；
	- **软件方式**：CPU 执行一条 `OUT` 指令，向控制端口写入特定值，经译码后生成启动脉冲。
	- **硬件方式**：由定时器（如 8253）或状态机自动产生周期性启动信号，实现**定时采样**。
- 转换结束后，ADC会输出**结束信号**，令主机获知**转换是否结束**，并进行**转换后数据的输入**：
	- 主机获取转换结束的方式有如下四种：
		- **查询方式**：CPU 不断读取 EOC 引脚状态，直到其变为“完成”状态。硬件简单，逻辑清晰。但CPU 效率低，一直在“等待”。
		- **中断方式**：将 EOC/INTR 信号连接到 **8259A 中断控制器** 的某条 IRQ 线，使用EOC/INTR 信号发送中断请求，在中断服务程序中读取转换后的值
		- **延时方式**：不使用转换结束信号，而是根据ADC的最大转换时间，使用软件延时或定时器等待足够时间后直接读取。
		- **DMA方式**：把结束信号作为DMA请求信号
``` asm
// 中断方式访问ADC
;数据段设置缓冲区
	adtemp  db 0  ;给定一个临时变量，用于暂存ADC转换结果
	
;代码段
	……            ;设置中断向量等工作
	sti           ;开中断
	
	mov dx,220h
	out dx,al     ;启动A/D转换
	……            ;其他工作
	
adint proc  ;中断服务程序
	sti  ;开中断
	
	;保护寄存器
	push ax   
	push dx
	push ds
	
	;设置数据段DS的段地址
	mov ax,@data  
	mov ds,ax
	
	; 读取ADC数据
	mov dx, 220h
	in al, dx          ; 从数据端口读取转换后的8位结果
	mov adtemp, al     ; 送入缓冲区
	
	; 发送EOI命令，提示中断已处理完
	mov al, 20h
	out 20h, al
	
	;恢复寄存器
	pop ds  
	pop dx
	pop ax
	iret  ;中断返回

adint endp
```

``` asm
// 使用查询方式对ADC0809的8个模拟通道依次进行A/D转换并获取数据
;数据段
	counter equ 8           ; 总共8个通道
	buf db counter dup(0)   ; 设立数据缓冲区

;代码段
	mov bx, offset buf      ; BX ← 缓冲区偏移地址
	mov cx, counter         ; CX ← 检测的数据个数
	mov dx, 220h            ; 从IN0开始转换

; 启动A/D转换
start1:  
	out dx,al               

; 循环查询是否转换结束
start2:  
	push dx
	mov dx,238h  
	in al,dx          ; 读入状态信息
	test al,80h       ; D7＝1，转换结束否?
	jz start2         ; 没有结束，则继续查询
	
	; 转换结束
	pop dx            
	in al,dx          ; 读取数据
	mov [bx],al       ; 存入缓冲区
	
	; 缓冲区和端口号递增，转向下一个模拟通道
	inc bx
	inc dx
	loop start1       ; CX = counter = 8,循环8次
	……                ; 数据处理
```
