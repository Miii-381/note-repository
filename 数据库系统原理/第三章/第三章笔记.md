### 以下是对**SELECT**语句后跟属性的总结表格：
#### **1. 基本属性**

| **属性/子句**  | **用途** | **语法示例**                                     | **说明**                     |
| ---------- | ------ | -------------------------------------------- | -------------------------- |
| `*`        | 选择所有列  | `SELECT * FROM table_name;`                  | 返回表中所有字段的数据（需谨慎使用，可能影响性能）。 |
| `列名1, 列名2` | 选择指定列  | `SELECT name, age FROM users;`               | 指定需要检索的字段，逗号分隔。            |
| `AS 别名`    | 字段别名   | `SELECT name AS full_name FROM users;`       | 为字段或表达式命名别名，提升可读性。         |
| `DISTINCT` | 去重     | `SELECT DISTINCT department FROM employees;` | 返回唯一值，去除重复记录。              |

#### **2. 条件过滤**

|**属性/子句**|**用途**|**语法示例**|**说明**|
|---|---|---|---|
|`WHERE 条件`|过滤行|`SELECT * FROM users WHERE age > 20;`|根据条件筛选数据（支持逻辑运算符 `AND`/`OR`）。|
|`BETWEEN 值1 AND 值2`|范围查询|`SELECT * FROM products WHERE price BETWEEN 100 AND 200;`|查询介于两个值之间的数据（含边界）。|
|`IN (值1, 值2)`|列表匹配|`SELECT * FROM users WHERE role IN ('admin', 'editor');`|匹配字段值在指定列表中的记录。|
|`LIKE '模式'`|模糊匹配|`SELECT * FROM users WHERE name LIKE 'J%';`|支持通配符 `%`（任意字符序列）和 `_`（单个字符）。|

#### **3. 分组与聚合**

|**属性/子句**|**用途**|**语法示例**|**说明**|
|---|---|---|---|
|`GROUP BY 列名`|分组统计|`SELECT department, COUNT(*) FROM employees GROUP BY department;`|按字段分组，常与聚合函数配合使用。|
|`HAVING 条件`|过滤分组|`SELECT department, COUNT(*) FROM employees GROUP BY department HAVING COUNT(*) > 5;`|对分组后的结果进行二次过滤。|
|**聚合函数**|计算汇总值|`SELECT SUM(salary), AVG(age) FROM employees;`|常用函数：`COUNT`、`SUM`、`AVG`、`MIN`、`MAX`。|

#### **4. 排序与限制**

|**属性/子句**|**用途**|**语法示例**|**说明**|
|---|---|---|---|
|`ORDER BY 列名 [ASC/DESC]`|排序|`SELECT * FROM users ORDER BY age DESC;`|默认升序（`ASC`），可指定降序（`DESC`）。|
|`LIMIT 数量`|限制返回行数|`SELECT * FROM users LIMIT 10;`|限制结果集的最大行数（部分数据库不支持，如 SQL Server）。|
|`OFFSET 偏移量`|跳过指定行|`SELECT * FROM users LIMIT 10 OFFSET 20;`|与 `LIMIT` 配合实现分页（如 `LIMIT 10 OFFSET 20` 表示跳过前20行，取10行）。|

#### **5. 连接与子查询**

| **属性/子句**       | **用途** | **语法示例**                                                                               | **说明**                                                |
| --------------- | ------ | -------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| `JOIN 表名 ON 条件` | 多表连接   | `SELECT users.name, orders.total FROM users JOIN orders ON users.id = orders.user_id;` | 通过 `JOIN` 关键字合并多个表的数据（支持 `INNER JOIN`、`LEFT JOIN` 等）。 |
| `子查询`           | 嵌套查询   | `SELECT * FROM users WHERE id IN (SELECT user_id FROM orders WHERE amount > 1000);`    | 在 `SELECT` 中嵌套另一个查询作为条件或数据源。                          |

#### **6. 其他高级用法**

|**属性/子句**|**用途**|**语法示例**|**说明**|
|---|---|---|---|
|`CASE WHEN 条件 THEN 值 END`|条件表达式|`SELECT name, CASE WHEN age < 18 THEN 'Minor' ELSE 'Adult' END AS status FROM users;`|根据条件动态生成字段值。|
|`UNION`|合并结果集|`SELECT name FROM users UNION SELECT name FROM customers;`|合并两个查询结果（自动去重，使用 `UNION ALL` 保留重复）。|

#### **示例总结**

##### **基础查询**
``` sql
SELECT id, name, age FROM users;
```

##### **带条件和排序**
``` sql
SELECT * FROM products WHERE price > 100 ORDER BY price DESC;
```

##### **分组与聚合**
``` sql
SELECT department, COUNT(*) AS employee_count 
FROM employees 
GROUP BY department 
HAVING COUNT(*) > 5;
```
    
##### **多表连接**
``` sql
SELECT users.name, orders.amount 
FROM users INNER JOIN orders ON users.id = orders.user_id;
```
    
##### **分页查询**
``` sql
SELECT * FROM articles ORDER BY created_at DESC LIMIT 10 OFFSET 20;
 ```

### **注意事项**

1. **性能优化**：避免使用 `SELECT *`，明确指定需要的列。
2. **去重**：`DISTINCT` 和 `GROUP BY` 都能去重，但 `GROUP BY` 通常用于分组统计。
3. **子查询**：嵌套查询需注意执行顺序和性能影响。

### **内连接、外连接、等值连接、自然连接** 的异同点总结表格：

---
#### **1. 核心定义与特点对比**

| **连接类型**               | **定义**                            | **特点**                                                             |
| ---------------------- | --------------------------------- | ------------------------------------------------------------------ |
| **内连接（INNER JOIN）**    | 基于指定条件合并两个表，**仅返回匹配的行**。          | - 默认连接类型，结果为两个表的**交集**。<br>- 不保留未匹配的行。                             |
| **外连接（OUTER JOIN）**    | 包括左外连接、右外连接、全外连接，**返回匹配行和未匹配的行**。 | - 左外连接保留左表所有行，右外连接保留右表所有行，全外连接保留所有行。<br>- 未匹配的字段用 `NULL` 填充。       |
| **等值连接（Equi Join）**    | 通过 **相等条件** 连接两个表，是内连接的一种特殊形式。    | - 条件仅使用 `=` 运算符。<br>- 可显式指定连接条件（如 `ON` 或 `WHERE`）。<br>- **保留重复列**。 |
| **自然连接（Natural Join）** | 一种特殊的等值连接，**自动匹配同名列并去重**。         | - 无需显式指定连接条件。<br>- 仅保留一次同名列。<br>- 要求连接列的**列名和数据类型必须一致**。           |

#### **2. 关键区别对比**

| **特性**   | **内连接（INNER JOIN）**                            | **外连接（OUTER JOIN）**                           | **等值连接（Equi Join）**                     | **自然连接（Natural Join）**            |
| -------- | ---------------------------------------------- | --------------------------------------------- | --------------------------------------- | --------------------------------- |
| **连接条件** | 显式指定（如 `ON` 或 `WHERE`）。                        | 显式指定（如 `LEFT JOIN ON`）。                       | 显式指定（如 `ON column1 = column2`）。         | 隐式匹配（自动识别同名列）。                    |
| **结果集**  | 仅包含匹配行（交集）。                                    | 包含匹配行和未匹配行（并集的一部分）。                           | 匹配行（交集），保留重复列。                          | 匹配行（交集），自动去除重复列。                  |
| **列处理**  | 保留所有字段（包括重复列）。                                 | 保留所有字段（包括重复列）。                                | 保留所有字段（包括重复列）。                          | 自动去除重复列（仅保留一次同名列）。                |
| **语法**   | `INNER JOIN ... ON`  或 `JOIN ... ON`。          | `LEFT JOIN` / `RIGHT JOIN` / `FULL JOIN`      | 通常通过 `INNER JOIN ... ON` 或 `WHERE` 实现。  | `NATURAL JOIN`                    |
| **灵活性**  | 最灵活（可自定义任意条件）。                                 | 灵活（支持左/右/全外连接）。                               | 灵活（仅限制于相等条件）。                           | 不灵活（依赖列名和数据类型）。                   |
| **示例**   | `SELECT * FROM A INNER JOIN B ON A.id = B.id;` | `SELECT * FROM A LEFT JOIN B ON A.id = B.id;` | `SELECT * FROM A, B WHERE A.id = B.id;` | `SELECT * FROM A NATURAL JOIN B;` |

#### **3. 相互关系**

- **内连接 vs 自然连接**：
    - **自然连接是内连接的特例**：自然连接本质上是内连接，但隐式匹配同名列并去重。
    - **内连接更通用**：内连接允许显式指定任意条件（包括非等值条件），而自然连接仅适用于同名列且等值条件。
- **等值连接 vs 内连接**：
    - **等值连接是内连接的一种形式**：内连接可以是等值连接（使用 `=`），也可以是非等值连接（使用 `>`、`<` 等）。
    - **等值连接的子集**：所有等值连接都是内连接，但内连接不一定是等值连接。
- **自然连接 vs 等值连接**：
    - **自然连接是等值连接的特例**：自然连接要求列名和数据类型一致，并自动去重，而等值连接只需满足等值条件，无需列名一致。

#### **4. 典型应用场景**

|**连接类型**|**适用场景**|
|---|---|
|**内连接**|查询两个表中**完全匹配的数据**（如有订单的客户）。|
|**外连接**|需要保留**左表/右表/所有表**的数据，即使无匹配（如查询所有员工及其部门）。|
|**等值连接**|显式指定等值条件，且需要保留重复列（如多表关联分析）。|
|**自然连接**|表结构清晰，列名一致，需简洁合并数据（如主键-外键关系）。|

#### **5. 注意事项**

- **自然连接的风险**：
    - 若表中存在多个同名列（如 `id` 和 `code`），自然连接会隐式匹配所有同名列，可能导致意外结果。
    - 数据类型不一致时，自然连接可能失败或返回错误。
- **等值连接 vs 自然连接**：
    - 自然连接适合列名和数据类型一致的场景，但**不推荐在复杂查询中使用**，因其隐式性可能导致维护困难。
    - 等值连接（显式 `ON` 条件）更安全可控，适合大多数场景。

#### 以下是 **INSERT** 关键字后的属性详细分类表格，基于 SQL 中常见用法和知识库中的相关内容整理：

---
##### **INSERT 关键字后的属性分类**

| **属性类别**        | **属性/关键字**                           | **说明与示例**                                                                               |
| --------------- | ------------------------------------ | --------------------------------------------------------------------------------------- |
| **基本语法**        | `INSERT INTO`                        | 指定向表中插入数据。<br>例如：`INSERT INTO table_name (column1, column2) VALUES (...);`              |
|                 | `VALUES`                             | 指定插入的具体值。<br>例如：`VALUES (value1, value2);`                                              |
| **列指定**         | `(column1, column2, ...)`            | 显式指定插入的列名，顺序需与值列表一致。<br>例如：`INSERT INTO users (name, age) VALUES (...);`                |
| **值处理**         | `DEFAULT`                            | 插入默认值（需列定义了默认值）。<br>例如：`INSERT INTO users (id) VALUES (DEFAULT);`                       |
|                 | `NULL`                               | 插入空值。<br>例如：`INSERT INTO users (email) VALUES (NULL);`                                  |
| **多行插入**        | 多个 `VALUES` 子句                       | 一次插入多行数据。<br>例如：`INSERT INTO users (name, age) VALUES ('Alice', 25), ('Bob', 30);`      |
| **大容量加载**       | `INSERT INTO SELECT`                 | 从其他表或查询结果中批量插入数据。<br>例如：`INSERT INTO new_table SELECT * FROM old_table;`                |
|                 | `TABLOCK` 提示（SQL Server）             | 提高大容量插入性能。<br>例如：`INSERT INTO new_table WITH (TABLOCK) SELECT * FROM old_table;`        |
| **表值参数**        | `@tvpNewCategories`（表值参数）            | 通过表值参数插入多行数据（需预定义类型）。<br>例如：`INSERT INTO Categories SELECT * FROM @tvp;`                |
| **默认值与自增列**     | `AUTO_INCREMENT` / `IDENTITY`        | 自动递增列（无需显式插入值）。<br>例如：`INSERT INTO users (name) VALUES ('John');`                       |
|                 | `SET IDENTITY_INSERT ON`（SQL Server） | 允许手动插入自增列的值。<br>例如：`SET IDENTITY_INSERT users ON; INSERT INTO users (id, name)...`      |
| **SELECT 模式转换** | `%SelectMode`（SQL Server）            | 控制数据输入的格式（逻辑/ODBC/显示模式）。<br>例如：`%SelectMode=2`（显示模式）。                                   |
| **条件插入**        | `INSERT INTO ... SELECT`（带 WHERE 条件） | 根据条件筛选数据插入。<br><br>例如：`INSERT INTO new_table SELECT * FROM old_table WHERE status = 1;` |
| **存储过程调用**      | `INSERT INTO ... EXECUTE`            | 调用存储过程插入数据。例如：`INSERT INTO logs EXEC usp_GetAuditData;`                                 |
| **特殊字符处理**      | 引号 `'` 或 `"`                         | 包裹字符串值。<br>例如：`INSERT INTO users (name) VALUES ('Alice');`                              |
| **列别名**         | 列名别名（在 `INSERT INTO` 中可省略，但需与值顺序一致）  | 例如：`INSERT INTO users (user_name, user_age) VALUES ('Alice', 25);`                      |

##### **关键属性对比示例**

|**属性**|**语法示例**|**适用场景**|
|---|---|---|
|**单行插入**|`INSERT INTO users (name, age) VALUES ('Alice', 25);`|插入单条记录。|
|**多行插入**|`INSERT INTO users (name, age) VALUES ('Alice', 25), ('Bob', 30);`|批量插入多条记录。|
|**默认值插入**|`INSERT INTO users (id) VALUES (DEFAULT);`|利用列的默认值自动填充。|
|**自增列插入**|`INSERT INTO users (name) VALUES ('John');`|自动递增列无需手动赋值。|
|**大容量加载**|`INSERT INTO new_table SELECT * FROM old_table;`|将数据从一个表迁移到另一个表。|
|**表值参数插入**|`INSERT INTO Categories SELECT * FROM @tvpNewCategories;`|通过预定义的表类型插入多行数据。|
|**存储过程调用**|`INSERT INTO logs EXEC usp_GetAuditData;`|调用存储过程生成数据并插入目标表。|

##### **注意事项**

1. **列顺序一致性**：`INSERT INTO` 后的列名顺序必须与 `VALUES` 中的值顺序一致。
2. **数据类型匹配**：插入的值类型需与目标列的数据类型兼容。
3. **主键/唯一约束**：插入数据时需避免违反主键或唯一约束。
4. **性能优化**：
    - 大批量插入时使用 `INSERT INTO SELECT` 或 `TABLOCK` 提示。
    - 避免频繁的单行插入，优先使用批量操作。
5. **事务控制**：对关键数据操作建议使用事务（`BEGIN TRANSACTION` / `COMMIT`）确保数据一致性。

### 更新（UPDATE）：
![[51.jpg]]

------------------
#### 以下是 **UPDATE** 关键字 的属性和用法整理表格，涵盖基本语法、高级用法、注意事项等内容：

##### **1. UPDATE 基本语法与属性**

| **属性类别**    | **属性/关键字**                               | **说明与示例**                                                                                                                                |
| ----------- | ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **基本语法**    | `UPDATE table_name`                      | 指定要更新的表。<br>例如：`UPDATE employees SET salary = 50000 WHERE id = 1;`                                                                       |
|             | `SET column1 = value1, column2 = value2` | 设置列的新值。<br>例如：`SET name = 'Alice', age = 30;`                                                                                            |
| **条件更新**    | `WHERE` 条件                               | 指定更新的行范围。<br>例如：`WHERE department_id = 5;`                                                                                               |
| **更新多列**    | 多个 `SET` 子句                              | 一次更新多个列。<br>例如：`SET salary = salary * 1.1, status = 'active';`                                                                           |
| **表达式更新**   | 使用算术表达式或函数                               | 例如：`SET price = price * 1.05;`（涨价5%）                                                                                                     |
| **子查询更新**   | `= (SELECT ...)`                         | 通过子查询设置新值。<br>例如：`SET department = (SELECT id FROM departments WHERE name = 'HR');`                                                      |
| **自增更新**    | `column = column + value`                | 例如：`SET stock = stock - 1;`（库存减少）                                                                                                        |
| **别名使用**    | `AS alias`                               | 为表或子查询设置别名。<br>例如：`UPDATE orders AS o SET o.status = 'shipped' WHERE o.id = 100;`                                                        |
| **JOIN 更新** | `UPDATE ... JOIN ... ON`（MySQL）          | 通过关联其他表更新数据。<br>例如：<br>`UPDATE employees e JOIN departments d ON e.dept_id = d.id SET e.salary = e.salary * 1.1 WHERE d.name = 'Sales';` |
| **默认值更新**   | `DEFAULT`                                | 设置列的默认值。<br>例如：`SET status = DEFAULT;`（需列定义了默认值）                                                                                         |
| **条件判断更新**  | `CASE WHEN ... THEN ... END`             | 条件分支更新。<br>例如：<br>`SET grade = CASE WHEN score >= 90 THEN 'A' WHEN score >= 80 THEN 'B' ELSE 'C' END;`                                   |
| **特殊操作**    | `LIMIT`（MySQL）                           | 限制更新行数。<br>例如：`UPDATE products SET price = 10 LIMIT 5;`                                                                                  |

##### **2. 高级用法与数据库特性**（除子查询之外基本不考）

| **属性**                 | **语法示例**                                                                                                                   | **适用场景**               |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------- | ---------------------- |
| **子查询更新**              | `UPDATE table SET column = (SELECT value FROM other_table WHERE ...);`                                                     | 从其他表动态获取值更新目标表。        |
| **JOIN 更新**            | `UPDATE employees e JOIN departments d ON e.dept_id = d.id SET e.salary = e.salary + 1000 WHERE d.name = 'Engineering';`   | 通过关联表条件更新数据（MySQL）。    |
| **CTE 更新**（PostgreSQL） | ` WITH updated AS (UPDATE employees SET salary = salary * 1.1 WHERE department_id = 5 RETURNING *) SELECT * FROM updated;` | 使用公共表表达式（CTE）更新并返回结果。  |
| **延迟更新**（MySQL）        | `DELAYED`（已弃用）                                                                                                             | 旧版本MySQL中用于异步更新（现不推荐）。 |
| **输出更新结果**（SQL Server） | `OUTPUT inserted.* INTO audit_table;`                                                                                      | 记录更新前后的数据到审计表。         |
| **锁机制**                | `WITH (ROWLOCK, UPDLOCK)`（SQL Server）                                                                                      | 控制并发更新时的锁粒度。           |

##### **3. 注意事项与常见问题**

|**注意事项**|**详细说明**|
|---|---|
|**WHERE 条件缺失**|不加 `WHERE` 会更新整张表的所有行，可能导致数据错误。|
|**数据类型兼容性**|更新值需与目标列的数据类型兼容，否则会报错或隐式转换（如截断）。|
|**主键/唯一约束冲突**|更新后可能导致主键或唯一索引冲突（如重复值），需提前检查。|
|**性能问题**|大批量更新时，考虑分批次更新或使用索引来加速条件匹配。|
|**事务控制**|对关键数据操作建议使用事务（`BEGIN TRANSACTION` / `COMMIT` / `ROLLBACK`）。|
|**锁竞争**|高并发更新时可能引发锁等待或死锁，需合理设计索引和事务隔离级别。|
|**触发器影响**|更新操作可能触发 `AFTER UPDATE` 或 `INSTEAD OF UPDATE` 触发器。|
|**自连接更新**|使用别名避免歧义，例如：`UPDATE table1 AS t1 JOIN table1 AS t2 ON ...`。|


##### **4. 典型示例**
###### **单表更新**
``` sql
-- 更新单列
UPDATE employees 
SET salary = 60000 
WHERE id = 101;

-- 更新多列
UPDATE products 
SET price = price * 1.05, stock = stock - 10 
WHERE category = 'Electronics';
```

###### **子查询更新**
``` sql
-- 从其他表获取值
UPDATE orders 
SET customer_id = (SELECT id FROM customers WHERE name = 'Acme Corp') 
WHERE order_number = 'ORD123';
```

###### **JOIN 更新（MySQL）**
``` sql
-- 通过关联部门表更新员工薪资
UPDATE employees e
JOIN departments d ON e.department_id = d.id
SET e.salary = e.salary + 500
WHERE d.name = 'Sales';
```

###### **条件分支更新（CASE）**
``` sql
-- 根据分数更新等级
UPDATE students
SET grade = CASE
    WHEN score >= 90 THEN 'A'
    WHEN score >= 80 THEN 'B'
    ELSE 'C'
END;
```

##### **5. 总结对比**

|**操作类型**|**适用场景**|**关键语法**|
|---|---|---|
|**简单更新**|单表、单列更新|`UPDATE table SET column = value WHERE condition;`|
|**多列更新**|同时修改多个字段|`SET column1 = value1, column2 = value2`|
|**条件更新**|动态计算或分支逻辑|`CASE WHEN ... THEN ... END`|
|**关联更新**|依赖其他表的数据|`JOIN`（MySQL）或 `CTE`（PostgreSQL）|
|**子查询更新**|从外部数据源获取值|`SET column = (SELECT ...)`|
|**性能优化**|大批量更新|分批次更新、添加索引、使用 `LIMIT`（MySQL）|

### 删除数据（DELETE）：

![[53.jpg]]

-----------
#### 以下是 **DELETE 关键字** 的属性和用法整理表格，涵盖基本语法、高级用法、注意事项及与其他命令的区别：

---
##### **1. DELETE 基本语法与属性**（数据库特有的属性不考）

| **属性类别**         | **属性/关键字**                               | **说明与示例**                                                                                                                                        |
| ---------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| **基本语法**         | `DELETE FROM table_name`                 | 指定要删除数据的表。<br>例如：`DELETE FROM employees WHERE id = 1;`                                                                                           |
|                  | `WHERE` 条件                               | 指定删除的行范围。<br>例如：`WHERE department_id = 5;`                                                                                                       |
| **删除所有记录**       | `DELETE FROM table_name;`（不加 `WHERE`）    | 删除表中所有数据，但保留表结构。<br>例如：`DELETE FROM logs;`                                                                                                       |
| **条件删除**         | `WHERE` 结合逻辑表达式                          | 例如：`WHERE salary < 3000 OR status = 'inactive';`                                                                                                 |
| **删除多表记录**       | `DELETE table1, table2 FROM ...`（MySQL）  | 删除多个表中的关联数据。<br>例如：`DELETE orders, customers FROM orders JOIN customers ON orders.customer_id = customers.id WHERE orders.status = 'cancelled';` |
| **子查询删除**        | `WHERE` 子句中嵌套子查询                         | 例如：`DELETE FROM employees WHERE department_id IN (SELECT id FROM departments WHERE name = 'HR');`                                                |
| **别名使用**         | `AS alias`                               | 为表设置别名。<br>例如：`DELETE e FROM employees AS e WHERE e.id = 100;`                                                                                   |
| **RETURNING 子句** | `RETURNING column1, column2`（PostgreSQL） | 返回被删除的数据。<br>例如：`DELETE FROM products WHERE id = 100 RETURNING name, price;`                                                                     |
| **级联删除**         | `ON DELETE CASCADE`                      | 删除主表记录时自动删除从表关联记录（需外键约束）。                                                                                                                        |
| **限制删除行数**       | `LIMIT`（MySQL）                           | 限制一次删除的行数。<br>例如：`DELETE FROM logs LIMIT 100;`                                                                                                   |

##### **2. 高级用法与数据库特性**（基本不考）

| **属性**                 | **语法示例**                                                                                             | **适用场景**              |
| ---------------------- | ---------------------------------------------------------------------------------------------------- | --------------------- |
| **JOIN 删除**（MySQL）     | `DELETE t1, t2 FROM table1 t1 JOIN table2 t2 ON t1.id = t2.id WHERE t1.status = 'invalid';`          | 通过关联多个表删除符合条件的数据。     |
| **CTE 删除**（PostgreSQL） | `WITH deleted AS (DELETE FROM employees WHERE department_id = 5 RETURNING *) SELECT * FROM deleted;` | 使用公共表表达式（CTE）删除并返回结果。 |
| **游标删除**（SQL Server）   | `DELETE FROM OPENROWSET(...)`                                                                        | 通过游标逐行删除数据。           |
| **事务控制**               | `BEGIN TRANSACTION ... ROLLBACK/COMMIT`                                                              | 在事务中执行删除操作，确保数据一致性。   |
| **错误处理**               | `TRY...CATCH`（SQL Server）                                                                            | 捕获删除操作中的异常（如约束冲突）。    |
| **优化器提示**              | `OPTION (MAXDOP 4)`（SQL Server）                                                                      | 控制并行删除的线程数。           |

##### **3. 注意事项与常见问题**

|**注意事项**|**详细说明**|
|---|---|
|**WHERE 条件缺失**|不加 `WHERE` 会删除整张表的所有行，可能导致数据永久丢失。|
|**外键约束冲突**|删除主表记录时，若从表存在关联记录，需先处理外键约束（如 `ON DELETE CASCADE`）。|
|**性能问题**|大批量删除时，考虑分批次删除或使用 `TRUNCATE`（清空表更快）。|
|**事务日志开销**|`DELETE` 会记录每行删除操作到事务日志，而 `TRUNCATE` 只记录页释放。|
|**索引与锁**|删除操作可能引发锁竞争或索引碎片，需合理设计索引和事务隔离级别。|
|**触发器影响**|删除操作可能触发 `AFTER DELETE` 或 `INSTEAD OF DELETE` 触发器。|
|**备份与恢复**|执行删除前务必备份数据，避免误操作导致不可逆损失。|

##### **4. DELETE 与 TRUNCATE/DROP 的区别**

|**特性**|**DELETE**|**TRUNCATE**|**DROP**|
|---|---|---|---|
|**作用范围**|删除特定行或所有行|删除所有行|删除整个表（结构+数据）|
|**速度**|较慢（逐行删除，记录事务日志）|快（直接释放数据页，少量日志）|极快（直接删除表对象）|
|**触发器**|触发 `DELETE` 触发器|不触发任何触发器|不触发触发器|
|**自增列重置**|不重置（如 MySQL 的 `AUTO_INCREMENT`）|重置（如 MySQL 的 `AUTO_INCREMENT`）|删除表后自增列也消失|
|**可回滚性**|在事务中可回滚|在事务中可回滚（部分数据库）|不可回滚（立即删除表）|
|**权限要求**|需 `DELETE` 权限|需 `ALTER` 或 `DELETE` 权限|需 `DROP` 权限|

##### **5. 典型示例**
###### **单表删除**
``` sql
-- 删除单行
DELETE FROM employees 
WHERE id = 101;

-- 删除多行
DELETE FROM products 
WHERE category = 'Outdated';
```

###### **子查询删除**
``` sql
-- 删除与子查询结果匹配的记录
DELETE FROM orders 
WHERE customer_id = (
    SELECT id FROM customers 
    WHERE name = 'Acme Corp'
);
```

###### **JOIN 删除（MySQL）**
``` sql
-- 删除关联订单和客户的记录
DELETE orders, customers 
FROM orders 
JOIN customers ON orders.customer_id = customers.id 
WHERE orders.status = 'Cancelled';
```

###### **CTE 删除（PostgreSQL）**
``` sql
-- 删除并返回结果
WITH deleted AS (
    DELETE FROM employees 
    WHERE department_id = 5 
    RETURNING *
)
SELECT * FROM deleted;
```

##### **6. 总结对比**

|**操作类型**|**适用场景**|**关键语法**|
|---|---|---|
|**简单删除**|单表、单条件删除|`DELETE FROM table WHERE condition;`|
|**多表删除**|跨表关联删除|`DELETE table1, table2 FROM ... JOIN ...`（MySQL）|
|**条件删除**|动态计算或复杂逻辑|`WHERE` + 子查询、`CASE` 表达式|
|**批量删除**|大数据量删除|分批次删除、`TRUNCATE` 替代方案|
|**安全删除**|避免误删|使用事务、`SELECT` 验证条件、`RETURNING` 确认结果|
|**级联删除**|外键关联表数据清理|`ON DELETE CASCADE`（需外键约束）|

##### **7. 数据库差异说明**

|**特性**|**MySQL**|**PostgreSQL**|**SQL Server**|
|---|---|---|---|
|**JOIN 删除**|支持 `DELETE table1, table2 ...`|支持 `DELETE FROM table1 ...`|不支持多表 `DELETE`|
|**RETURNING 子句**|不支持|支持|不支持|
|**TRUNCATE 与权限**|需 `DELETE` 权限|需 `TRUNCATE` 权限|需 `ALTER` 权限|
|**自增列重置**|`TRUNCATE` 重置|`TRUNCATE` 重置|`TRUNCATE` 重置|
