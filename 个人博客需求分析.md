# 技术栈：
- SSM三件套后端+Vue3网页前端（可能会搞搞小程序）
# 需求分析：
1. **CRUD**：对文章的增删改查
2. 用户分级：
	- **系统管理员**：最高权限，可触及前后端所有接口和数据（**使用SQL内置权限管理，直接控制底层数据库**）
	- **内容管理员**：对文章和普通用户进行管理
	- **普通用户**：可个性化账号，可查找文章（精确和模糊查找，条件查找），可对自己的文章进行增删改查，可对别人的文章进行点赞和评论（**上面俩使用后端代码自定义权限控制**）
3. 登录：
	- 账号密码登录（或者用户名密码登录，最好合到一起，合并不了就用个下拉框切换API）
	- 手机号验证登录（接api似乎有点麻烦，发短信多半还要钱）
	- 邮箱验证登录（SMTP倒是挺简单）
	- 第三方验证登录（QQ，微信，到时候看看API难不难）
	- **随机验证码！！！！！！！！！！**（接个reCAPTCHA应该可行）
4. 具体功能：
	- **文章导入导出和在线编辑（基本功能）**：准备完全使用markdown格式，md解析器插件暂定为vue-markdown，富文本编辑器暂定为md-editor-v3，文章导入导出直接上代码就行
	- **用户注册和登录（基本功能）**：注册用户均为普通用户，自带一个管理员账户，通过管理员账户进行权限分配。注册需要进行人机验证，可以接reCAPTCHA或者其他验证码API。
	- **文章点赞、评论、转发**：点赞挺简单，评论也可以用编辑器，转发发个域名就行。
5. 数据库：
	- 库（PersonalBlog）
	- 表：
		- 用户信息表（==用户ID==，==用户名==，==邮箱==，第三方验证字段{没想好，感觉可以拆成单独的表}，==密码{尽量存哈希值}==，==创建时间，最后登录时间==，==用户状态{正常、审核、封禁、删除、纪念}==，==用户头像地址==，用户简介）
		- 角色表（角色ID，角色名，角色描述）
		- 权限（权限ID，权限名，权限描述）
		- 角色-权限关联表（角色ID，权限ID）

		- 操作日志表（日志ID，用户ID，操作类型，操作详情，操作时间）

		- 文章（文章ID，用户ID，文章标题，文章摘要，文章状态{编辑、审核、通过、下架}，是否允许评论，物理存放地址{相对地址，后期拼接}，缩略图存放地址，创建时间，更新时间）
		- 文章统计（文章统计ID，文章ID，点赞数，收藏数，浏览量，评论量）
		- 文章标签（标签ID，类型名，类型描述）
		- 文章-标签关联表（文章ID，标签ID）
		- 收藏记录（收藏ID，用户ID，文章ID，收藏时间）
		- 点赞记录（点赞ID，用户ID，文章ID，点赞时间）
		- 评论（评论ID，文章ID，用户ID，评论内容，父评论ID，评论状态，创建时间，用户IP地址）
		- 通知（通知ID，用户ID，通知类型，关联操作的ID（具体是哪种操作引起的通知），是否已读）
（权限控制使用RBAC方式进行，权限控制直接连接四个表就行，最多做做语法树优化，这个水平就别考虑高并发场景了......）
（还是使用传统的外键方式进行级联控制，懒得写控制代码了，太繁琐了......）
