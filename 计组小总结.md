# 原码、反码、补码、移码
- 正数原码、反码、补码都一样
- 负数原码除符号位取反得反码，除了符号位取反+1（反码+1）得补码。
- 正负数的移码均为补码的符号位取反
- **快速求补码**：找到从数值位的右边往左边的第一个1，这个1左边除了符号位全部取反，右边包括这个1都不变，做完之后+1
# 求定点小数
- 出题一般出的都是2的次幂，很少出现循环小数，尤其是分数形式求原反补码。出现了截断就行。
- **2的次幂为分母的真分数如何简单化为定点小数：
	- 先把分子的二进制求出来。比如23对应10111
	- 再把分母的二进制**位数**算出来，比如32就是2的6次方，小数点后有6位，可用0.000000表示
	- 将分子填入分母，分子的最高位对齐分母的最高位。则此时23/32 = 0.1011100（8位定点数)
# IEEE754浮点数：
- 32位单精度浮点数：1位符号位，8位阶码（移码存储，$E=e + 127$），23位尾数（原码）
- 64位双精度浮点数：1位符号位，11位阶码（移码存储，$E=e + 1023$），52位尾数（原码）
- 考试一般考单精度浮点数。
- **十进制小数规格化为IEEE754浮点数方法**：
	- **确定符号位**：1为负，0为正。
	- **将小数转化为二进制**
	- **规范化**：将转化后的二进制数表示为 $1.xx \times 2^e$ 的形式。左移指数为负，右移指数为正，**对任何小数都适用，不局限于小于1的小数**。
	- **计算偏移后的指数**：$E=e + 127$
	- **组合最终结果**：| 符号位 | 指数 | 尾数 |
- 反过来化也一样。

# 浮点运算：
## 加减法：
1. **0 操作数检查**：若其中一个操作数为 0，直接返回另一个操作数，避免后续复杂计算。

2. **对阶（Align Exponents）**：
	- **目的**：使两个浮点数的阶码相同，便于尾数加减。
	- **操作**：
	    - 比较两个浮点数的阶码（指数部分），**较小的阶码向大的阶码对齐。**
	    - **尾数右移**（而非左移），以减少精度损失。每右移一位，阶码加 1。
	    - 例如：若 $x=1.23×10^3$，$y=3.45×10^2$，需将 $y$ 转换为 $0.345×10^3$。

3. **尾数加减（Mantissa Addition/Subtraction）**
	- **符号位处理**：根据符号位决定加法或减法。
	    - 若符号相同，尾数相加。
	    - 若符号不同，尾数相减。
	- **双符号位补码运算**：用于检测溢出（如 `01` 或 `10` 表示溢出）。

4. **结果规格化（Normalization）**：
	- **目的**：确保尾数的最高有效位为 1（规范化形式）。
	- **操作**：
	    - **右规（Right Shift）**：若尾数溢出（如 `01×××` 或 `10×××`），右移一位，阶码加 1。
	    - **左规（Left Shift）**：若尾数下溢（如 `00×××` 或 `11×××`），左移直到最高位为 1，阶码减相应位数。

5. **舍入处理（Rounding）**
	- **目的**：保留有效位数，减少误差。
	- **方法**：
	    - **0 舍 1 入法**：若移出的低位全为 0，舍弃；若最低位为 1，进位。
	    - **恒置 1 法**：无论移出的低位如何，最低位恒置 1（简化硬件实现）。

6. **溢出判断（Overflow/Underflow）**
	- **阶码溢出**：若阶码超出表示范围，置为无穷大（上溢）或零（下溢）。
	- **尾数溢出**：通过规格化处理解决。
## 乘除法：
1. **0 操作数检查**：若有一个操作数为 0，直接返回 0。

2. **符号位处理**：
- **乘法**：结果符号位为两数符号位的异或（正负由异或决定）。
- **除法**：结果符号位为被除数与除数符号位的异或。

3. **阶码运算**：
	- **乘法**：$结果 = 阶码A + 阶码B - 偏移量(127或1023)$
	- **除法**：$结果 = 阶码A - 阶码B$

4. **尾数运算**
	- **乘法**：$结果 = 阶码A \times 阶码B$
	- **除法**：$结果 = 阶码A \div 阶码B$

5. **结果规格化**
	- **右规**：若尾数溢出（如 `01×××` 或 `10×××`），右移一位，阶码加 1。
	- **左规**：若尾数下溢，左移直到最高位为 1，阶码减相应位数。

6. **舍入处理**：与加减法相同，保留有效位数，减少误差。

7. **溢出判断**：阶码溢出或尾数溢出时，按 IEEE 754 规则处理（如置无穷大或零）。
