# 1. 可见性/遮挡——油画家算法及深度缓冲算法（Z-Buffer）
- 对于图形的绘制，人们通过观察画家作画的顺序，提出了“**油画家算法”（Painter‘s algorithm）**
![[改图鸭2.png]]![[改图鸭3.png]]
- 油画家算法没有深度信息，纯靠手动对图形按照"从远到近“的顺序进行排序并绘制，来解决图形间的遮挡问题，很难处理复杂的遮挡关系（如上图的 Penrose 三角形）。
- 在 3D 游戏中，物体的位置是不断变化的。如果我们每帧都要手动对成千上万个三角形进行“从远到近”的排序，计算量会非常巨大且复杂。
- 为了解决这两个问题，我们引入了**深度缓冲算法（Z-Buffer）**——通过局部深度比较，彻底摆脱了对绘制顺序的依赖。

![[改图鸭4.png]]
## Z-Buffer 简介（有请 AI 大佬入场~ ）：
- 计算机在渲染时，会维护两张“地图”（缓冲区）：
	1. **帧缓冲（Color Buffer / Frame Buffer）**：存储每个像素的**颜色**（比如这个点是红色的）。
	2. **深度缓冲（Depth Buffer / Z-Buffer）**：存储每个像素的**深度值（Z 值）**，即该像素点距离摄像机的距离。
- 核心逻辑（更新算法）：
	- 当计算机准备在一个像素点 `(x,y)`绘制一个新颜色时，它会进行以下对比：
		1. 计算新物体在这个点上的深度值 $Z_{new}​$；
		2. 读取深度缓冲中该点已有的深度值 $Z_{old}​$；
	    3. **判断：** 如果 $Z_{new}<Z_{old}$（新点更近）：说明新物体挡住了旧物体。于是**更新**颜色缓冲为新颜色，并把深度缓冲里的值更新为 $Z_{new}​$；如果$Z_{new}>Z_{old}$（新点更远）：说明新物体在旧物体的后面，被挡住了。于是**丢弃**这个点，不做任何修改。
- **优点：**
    - **简单且高效**：逻辑非常简单，极其适合硬件（显卡 GPU）并行加速。
    - **与顺序无关**：无论你先画近的物体还是先画远的物体，最终屏幕上的结果都是正确的（这大大简化了渲染引擎的设计）。
    - **硬件友好**：深度测试可并行化，适合 GPU 流水线（Rasterizer → Early-Z → Shader → Late-Z）。
- **缺点：**
    - **内存开销**：需要额外的显存来存储深度信息。深度缓冲通常占显存 25%~30%（如 1080p + 24-bit depth = ~8MB）；
    - **Z-Fighting（深度冲突）**：当两个平面靠得非常非常近时，由于浮点数精度限制，计算机会分不清谁前谁后，导致比较结果在帧间随机翻转，使得画面出现闪烁的斑点。
    - **无法处理半透明**：Z-Buffer 只能处理“全遮挡”。对于半透明物体（如玻璃），通常仍需要从远到近的手动排序。

![[Pasted image 20260409172333.png]]
- 在绘制画面时，计算机需要存放两张图片，一张是渲染画面，一张是深度图，对应了 Z-Buffer 的两个缓冲区。
	- 深度图是一张**灰度图**，颜色**越深**，物体离视口**越近**；反之则越远。

- **回答 Complexity 内的问题：**
	- 这个算法并没有排序，只是遍历了所有三角形的像素，寻找对应像素的最小值并替换，相当于提取最小值，对每个三角形，仅遍历其覆盖的像素（或子采样点），对每个采样点执行 $O(1)$ 深度比较。因此总时间复杂度为 $O(N)$，其中 `N` 是所有三角形覆盖的采样点总数（`≈ 像素 × 平均覆盖率`），与三角形数量 `n` 无关。。
- **回答中间的问题：**
	- 这个算法有个优良的性质：**它的最终结果跟绘制顺序没关系**。理论上，不管图形怎么绘制，只要深度值是对的，深度计算算法是对的，那么最后得到的深度图一定是同一个结果。
	- 但，理想很丰满，计算机存储坐标时，用的基本都是浮点数，浮点数肯定有精度限制和比较误差，这时就会产生**深度冲突（Z-Fighting）** 问题。解决方案有很多，比如对其中一个表面施加微小的深度偏移（偏移多边形）、在模型设计时就避免几何体重叠、提高深度缓冲精度等等方式。这里暂时不做延申。
- Z-Buffer 是现代实时渲染的基石：它以极低的算法复杂度（O(n)）和硬件可并行性，解决了可见性判定这一核心难题。它已经成为了行业内最重要、使用最广泛的遮挡计算方式，至今仍是所有 GPU 固定管线与现代光栅化器的默认方案。

- 这就要介绍一下计算机图形学中的**着色（Shading）**概念了
- 着色（Shading）是计算机图形学中的一个核心概念，指的是**为三维场景中的表面计算颜色和亮度的过程**，目的是让渲染出的图像看起来更真实或具有特定的艺术风格。
- 简而言之，**着色 = 根据光照、材质和几何信息，决定每个像素的颜色**
![[改图鸭8.png]]
- **注**：“applying a material” 强调的是**为几何体指定表面属性**，如颜色、反射率、粗糙度等，是计算机图形学中“着色”的核心含义。
	- 在计算机图形学中，**材质（Material）** 是描述**物体表面如何与光线交互**的一组属性。它决定了物体在光照下的视觉外观，比如颜色、光泽度、透明度等。
	- 材质会告诉渲染系统： “当光线照射到这个表面时，它应该反射多少光？以什么方式反射？”
	- **材质 = 物体表面的“光学身份证”**，定义了它看起来是木头、金属、玻璃还是布料。

- Blinn-Phone 反射模型包括以下三种光线成分：
	1. **环境光（Ambient）**
	    - 模拟全局间接光照，即光线在场景中多次反弹后产生的微弱均匀照明。
	    - 不依赖于光源方向或观察方向，通常表示为常量
	2. **漫反射（Diffuse）**
	    - 模拟粗糙表面对光的均匀散射（遵循 Lambert 定律）。
	    - 依赖于**入射光方向** $\vec{l}$ 与**表面法向量** $\vec{n}$ 的夹角：
	3. **镜面反射（Specular）**
	    - 模拟光滑表面的高光（如金属、塑料的亮斑）。
	    - **Blinn-Phong 的关键改进**：使用**半角向量** $\vec{h}=\frac{\vec{l}+\vec{v}}{||\vec{l}+\vec{v}||}$（ $\vec{v}$ 为视线方向），计算 $\vec{h}$ 与法向量 $\vec{n}$ 的夹角，而非 Phong 模型中的反射向量 $\vec{r}$与 $\vec{v}$ 的夹角。

- 表面法向量：着色点视作一个微平面，表面法向量垂直于当前着色点

- 解释一下 “Shading is Local”：
	- **“着色具有局部性”（Shading is Local）** 是计算机图形学中的一个核心概念，意思是：在计算某个表面上某一点的颜色（即“着色”）时，**只依赖于该点附近的局部信息**，而**不考虑**整个场景的**全局光照传播**（如多次反射、折射等复杂光路）。
	- 由于在传统的实时渲染或基础光照模型（如 Phong、Blinn-Phong）中，计算一个**着色点**（shading point）的颜色时，仅使用上述的4种**局部信息**，这些信息都**集中在该点本身或直接相关的方向上**，不涉及：
		- 其他物体如何反射光到该点（间接光照）
		- 光线在场景中多次弹射（全局光照）
		- 阴影是否被其他物体遮挡（除非额外计算阴影贴图）
	- 所以其不会产生诸如阴影等具有全局性的渲染结果。这是实时图形（如游戏）高效渲染的基础，但也是其真实感受限的原因之一。

	- 想象一个红色球放在白色地板上，旁边有一盏白光：
		- **局部着色**：球的底部是黑的（因为没有直射光），不会呈现红色。
		- **全局光照**：地板会接收到球反射的红光，底部微微泛红（颜色溢出，Color Bleeding）。
	- 局部着色**看不到全局光照那种环境光效果**，因为它“只关心自己这点有没有被光照到”，不关心“别的地方的光会不会照过来”。

- **朗伯（Lambert）漫反射公式**用于计算物体表面因**漫反射**而呈现的亮度，用“法线和光方向的夹角余弦”来决定表面有多亮
- 它的核心思想是：粗糙表面将入射光**均匀地向所有方向散射**，且亮度只与**光线入射角**有关，与观察方向无关。
- 入射角越小（光线越垂直于平面），表面越亮；斜射则变暗。
- $\text{max}(0, n \cdot l)$ 用于确保背面不被照亮（当 θ > 90° 时，cosθ 为负，点积即为负，此时设为0）

- 漫反射系数越大，反射光线的能力越强，在同样的光源下表面反射的光线越多，物体看起来就越亮：
